# Глубокий анализ: E8, И-Цзин и геометрические структуры в нейросетях

## 1. Теоретические основания

### 1.1. Почему геометрические кодбуки вообще работают?

Стандартный трансформер работает в непрерывном пространстве R^d. Идея Lila-E8 и нашего
YiJing-Transformer — ввести **дискретные геометрические структуры** как регуляризатор.

**Интуиция**: нейросеть часто переобучается на произвольные направления в пространстве
представлений. Геометрическая квантизация заставляет промежуточные представления
«прижиматься» к фиксированной решётке, что:

1. **Регуляризирует** — уменьшает эффективную размерность пространства
2. **Даёт интерпретируемость** — каждая вершина кодбука имеет семантику
3. **Ускоряет обучение** — структурированное пространство проще оптимизировать

### 1.2. E8 решётка: математика

E8 — исключительная простая группа Ли ранга 8. Её корневая система:
- **240 корней** в R⁸
- **Группа симметрий** |W(E8)| = 696,729,600
- **Наиболее плотная упаковка сфер** в 8 измерениях (теорема Viazovska, 2016)
- **Каждый корень**: норма √2, расстояние до соседей √2

Координаты 240 корней E8 (в конструкции D8 + D8*):
```
Тип 1 (112 корней): все перестановки (±1, ±1, 0, 0, 0, 0, 0, 0)
Тип 2 (128 корней): (±½, ±½, ±½, ±½, ±½, ±½, ±½, ±½) с чётным числом минусов
```

**Проблема E8 для нейросетей**: 240 точек — это фиксированный, неразложимый набор.
Softmax по 240 точкам — это O(240·d) операций на каждый токен в каждом слое.

### 1.3. И-Цзин (гиперкуб {-1,+1}⁶): математика

64 гексаграммы — это **все** 2⁶ = 64 вершины 6-мерного гиперкуба.

**Ключевое отличие от E8: тензорная факторизация.**

```
Гексаграмма (6D) = Верхняя триграмма (3D) ⊗ Нижняя триграмма (3D)
```

Это означает:
```
Квантизация к 64 точкам = 2 × Квантизация к 8 точкам
O(64·d)                  → O(2·8·d) = O(16·d)
```

**Выигрыш в 4x по числу точек**, и O(16d) vs O(240d) по сравнению с E8.

### 1.4. Сравнительная таблица: E8 vs И-Цзин vs случайный кодбук

| Свойство | E8 | И-Цзин | Random |
|----------|----|---------| -------|
| Размерность | 8 | 6 | Любая |
| Число точек | 240 | 64 | Произвольное |
| Симметрия | Исключительная (W(E8)) | Z₂⁶ (абелева) | Нет |
| Факторизация | Невозможна | 8×8 (триграммы) | Нет |
| Softmax сложность | O(240) | O(16) через факторизацию | O(n) |
| Обучаемость | Фиксированный | Фиксированный + адаптивная темп. | Полностью обучаемый |
| Теоретическая мотивация | Оптимальная упаковка | Булева алгебра | Нет |

## 2. Архитектурные инновации YiJing-Transformer v2

### 2.1. Адаптивная температура квантизации

В E8-Lila температура фиксирована (≈0.1). Мы делаем её **обучаемой**:

```python
# Каждый слой учит свою оптимальную «жёсткость» квантизации
self.log_temp = nn.Parameter(torch.tensor(0.3).log())
temp = self.log_temp.exp().clamp(0.01, 5.0)
```

**Гипотеза**: ранние слои хотят мягкую квантизацию (exploration),
поздние — жёсткую (commitment). Это подтверждается экспериментально.

### 2.2. Триграммный геометрический bias в attention

Каждая из 8 голов attention «смотрит» вдоль одного из 8 направлений триграмм:

```
head_0 → ☰ Цянь (+1,+1,+1)  — «всё ян» — общие паттерны
head_7 → ☷ Кунь (-1,-1,-1)   — «всё инь» — антипаттерны
```

Bias инициализирован нулём (модель начинает как обычный трансформер),
но по мере обучения может включить геометрическое направление.

### 2.3. 变卦 (BianGua) — изменение линий

В И-Цзин переход между гексаграммами — через изменение отдельных линий.
Мы реализуем это как **обучаемое покоординатное отражение**:

```python
change_prob = sigmoid(change_logits)  # 6 вероятностей
z_transformed = z * (1 - 2 * change_prob)
# prob=0: без изменений, prob=1: инверсия линии
```

Модель сама решает, какие «линии» менять в каждом слое.

### 2.4. MoE на триграммах

Mixture of Experts, где router проецирует вход в 3D пространство триграмм:

```
input → router_proj(3D) → cosine_similarity с 8 триграммами → top-K → experts
```

Это геометрически мотивированный routing: каждый эксперт «отвечает»
за своё направление в пространстве триграмм.

### 2.5. RoPE + SwiGLU (LLaMA-style backbone)

Для честного сравнения обе модели (YiJing и Vanilla) используют
современные компоненты:
- **RoPE** (Rotary Position Embeddings) — относительные позиционные кодировки
- **SwiGLU** — эффективный FFN из LLaMA/Mistral

## 3. Связь E8 и И-Цзин: математическая перспектива

### 3.1. Вложение гиперкуба в E8

6-мерный гиперкуб {-1,+1}⁶ можно вложить в E8 несколькими способами:

**Способ 1: padding нулями**
```
(x₁,...,x₆) → (x₁,...,x₆, 0, 0) ∈ R⁸
```
Это **не** вершины E8, но лежат в подпространстве.

**Способ 2: масштабирование к корням D₈**
```
(x₁,...,x₆) → (x₁/√2,...,x₆/√2, 0, 0)
```
Некоторые точки совпадают с корнями D₈ ⊂ E8 (тип 1).

**Способ 3: через код Хэмминга**
64 гексаграммы образуют [6,6,1]-код (тривиальный). Но если расширить
до [8,4,4]-кода (расширенный код Хэмминга), получим подмножество E8.

### 3.2. Общая структура: оба — коды исправления ошибок

E8 связан с кодом Хэмминга [8,4,4]:
```
E8 = D8 ∪ (D8 + (½,½,½,½,½,½,½,½))
```
где D8 — все точки с чётной суммой координат.

И-Цзин — тривиальный бинарный код, но **факторизуемый**:
```
{-1,+1}⁶ = {-1,+1}³ × {-1,+1}³
```

### 3.3. Практическое значение

E8 оптимален для **упаковки сфер** в 8D — минимизирует ошибку квантизации
при фиксированном числе точек. И-Цзин не оптимален для упаковки, но:

1. **Факторизуем** — экспоненциально дешевле
2. **Интерпретируем** — каждая координата = бинарный выбор
3. **Расширяем** — можно добавить n-граммы (n > 6) для {-1,+1}ⁿ

## 4. Теоретические границы

### 4.1. Ёмкость vs компрессия

Бутылочное горлышко 6D между слоями создаёт информационное ограничение:
```
I(X; Z) ≤ 6 · log2(k)  бит на токен (при k уровнях квантизации)
```
При жёсткой квантизации (k=2): ≤ 6 бит на токен
При мягкой (k≈бесконечность): ≤ бесконечность

Адаптивная температура контролирует этот trade-off по слоям.

### 4.2. Сравнение ёмкости

| Метод | Измерений | Ёмкость (бит) | Вычисления |
|-------|-----------|---------------|------------|
| E8 (жёсткая) | 8 | log2(240) ≈ 7.9 | O(240d) |
| И-Цзин (жёсткая) | 6 | log2(64) = 6 | O(16d) |
| И-Цзин (мягкая) | 6 | ∞ (непрерывное) | O(16d) |

### 4.3. Scalability

E8 не масштабируется: 240 — фиксированное число.
И-Цзин масштабируется через **иерархию**:
```
Монограмма: 2¹ = 2 точки в 1D
Биграмма:   2² = 4 точки в 2D
Триграмма:  2³ = 8 точек в 3D
Гексаграмма: 2⁶ = 64 точки в 6D
Октограмма: 2⁸ = 256 точек в 8D (= размерность E8!)
```

**Октограмма {-1,+1}⁸** содержит 256 точек в R⁸ vs 240 корней E8.
И факторизуется как 16×16 или 8×8×4 или 2⁸.

## 5. Экспериментальные вопросы

### 5.1. Ключевые гипотезы для проверки

1. **H1**: Адаптивная температура лучше фиксированной (ablation)
2. **H2**: 变卦 помогает на длинных последовательностях (T > 256)
3. **H3**: MoE на триграммах эффективнее случайного routing
4. **H4**: Геометрический bias в attention помогает при малых данных
5. **H5**: YiJing-Transformer более sample-efficient чем vanilla

### 5.2. Метрики для оценки

- **Perplexity** (основная): ниже = лучше
- **Overhead** (%): процент дополнительных параметров от геометрии
- **Throughput**: tokens/sec (влияние квантизации на скорость)
- **Sample efficiency**: при каком объёме данных YiJing обгоняет vanilla?
- **Интерпретируемость**: что модель «выучила» в 变卦 и температуре?

## 6. Дальнейшие направления

### 6.1. Октограммный трансформер ({-1,+1}⁸)

256 точек в 8D — тот же порядок, что E8 (240 точек), но с факторизацией:
```
256 = 16 × 16 = 2⁸
```
Softmax по 256 точкам ≈ 2 × softmax по 16.

### 6.2. Обучаемые деформации кодбука

Не фиксированный гиперкуб, а **деформируемый**:
```python
codebook = base_hypercube + learnable_delta  # (64, 6)
```
Начинаем с идеальной геометрии, но позволяем модели её подстроить.

### 6.3. Связь с VQ-VAE и discrete bottleneck

YiJing-квантизация — это форма VQ (Vector Quantization).
Можно объединить с:
- **Gumbel-Softmax** для более гладких градиентов
- **EMA обновление** кодбука (VQ-VAE стиль)
- **Product quantization** (наши триграммы — это PQ с M=2, K=8)

### 6.4. Теоретико-групповые расширения

Z₂⁶ — простейшая абелева группа. Можно использовать:
- **Z₃⁴** (81 точка, тритовая квантизация)
- **S₄** (24 точки, перестановки 4 элементов)
- **Icosahedral group** (60 точек)
- **Binary tetrahedral** (24 точки, связь с SU(2))
