## 1. Концептуальная стадия

### 1.1. Ядро идеи — тождество, не метафора

```
64 гексаграммы И-Цзин  ≅  вершины гиперкуба Q6 = {-1,+1}⁶
                           ↓
                     Коды Рида-Маллера  (линейный код d=2)
                           ↓
                     Уолш-Адамар WHT    (O(n log n) спектр)
```

Это не аналогия — это математическое тождество. Структура существовала до проекта; задача — научить трансформер ею пользоваться.

### 1.2. Выбор Варианта 3

Проект прошёл развилку трёх путей:

| Вариант | Описание | Статус |
|---------|----------|--------|
| Вариант 1 | Исследовательская архитектура (PPL-гонка) | Отклонён |
| Вариант 2 | Практический мультидоменный агент (NautilusMoME) | Отклонён |
| **Вариант 3** | **Культурно-математическая система** | **Выбран** |

**Девиз Варианта 3:** *«Трансформер, который думает гексаграммами и говорит словами.»*

Архетип — не слой модели, а **сквозная ось** всей архитектуры.

| Источник | Вклад | Реализация | Эффект (ablation) |
|----------|-------|------------|-------------------|
| **Скляров** | И-Цзин как вычислительная машина; дворцовая кластеризация | PalaceAttention, архетипный кодбук | -0.28 PPL (вреден изолированно) |

### 1.4. Система знаний (4 уровня)

```
Формула  →  Архетип  →  Алгоритм  →  Теорема
```

19 теорем формально доказаны и задокументированы (см. `theoretical-foundations.md`).

### 1.5. Тернарная логика — третье измерение

```
Двоичная:  {0, 1}      — «да» / «нет»
Тернарная: {-1, 0, +1} — «нет» / «в переходе» / «да»
```

В И-Цзин это 变爻 (меняющаяся линия) — линия в процессе перехода, не инь и не ян. В нейросетевом смысле: третье состояние фиксирует момент семантического сдвига.

### 2.4. Ключевой фикс: STE zero-gradient trap

**Проблема:** При T→0 (жёсткая квантизация) градиенты через STE обнулялись; 64 архетипа никогда не активировались.

**Решение:** Temperature annealing: T: 1.0 → 0.1 за 2000 шагов.

```
До фикса:   active_archetypes = 0 навсегда
После фикса: active_archetypes = 64 (к шагу 200)
             archetype_usage_mean = 0.703
```

### 2.5. Текущая открытая проблема: разрыв Bridge vs Interlingua

```
Лучший bridge (v59):      PPL 1.24
Лучшая интерлингва (v61): PPL 2.92
Vanilla (baseline):       PPL 2.94
─────────────────────────────────
Вклад интерлингвы:    -0.02 PPL  (= статистический шум)
Вклад мостов:         -1.70 PPL  (= существенный эффект)
```

**Корневая причина:** Общий `trit_proj` для всех модулей → все модули проецируются через одну матрицу → триты одинаковые → 64 архетипа неразличимы → readout attention видит 64 копии → выход интерлингвы = бесполезная константа → gate стабилизируется на ~0.49.

**Диагностика:** `global_gate` = 0.489–0.494 во всех конфигурациях. Инициализация 0.5, сдвиг -0.01 означает: модель слегка избегает интерлингву, не находя в ней сигнала.

## 3. Нерешённые вопросы

### Технические

1. **Разрыв Bridge↔Interlingua:** Почему hub-and-spoke через архетипы не работает, хотя теоретически должен?
   - Гипотеза: недостаточная дифференциация тритов (per-source trit_proj решит?)
   - Вероятность фикса вариантом B (per-source): 60%

2. **800 шагов < 2000 warmup:** Temperature annealing не завершён; жёсткая фаза (T→0.1) не достигнута.
   - Нужна валидация на 2000+ шагах

3. **Synthetic vs real data:** Все эксперименты на синтетическом WikiText. Поведение на реальных корпусах неизвестно.

### Концептуальные

4. **5 источников ≈ вредны:** На 800 шагах Скляров, Фомюк, Андреев, Касаткин дают негативный вклад. Противоречит теоретическому обоснованию — нужен эксперимент на длинном обучении.

5. **Interlingua vs Bridge:** Глубокая теоретическая разница или engineering artifact?
