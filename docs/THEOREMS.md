# 19 теорем YiJing-Transformer: полный реестр

Документ фиксирует все 19 теорем, заявленных в KNOWLEDGE_FRAMEWORK.md.
7 из них полностью верифицированы вычислительными экспериментами в
`scripts/theoretical_analysis.py`. Остальные 12 имеют формулировки,
но без формального доказательства или программной верификации.

---

## Теоремы с вычислительной верификацией (7/19)

### Т1. XOR-изоморфизм

**Утверждение:** Покоординатное умножение в {-1,+1}^n изоморфно XOR в Z_2^n.
Отображение psi(b) = (-1)^b — гомоморфизм (Z_2, XOR) -> ({-1,+1}, *).

**Верификация:** `scripts/theoretical_analysis.py::theorem_1_xor_isomorphism()`
- Проверена замкнутость для всех 8x8 пар триграмм
- Построена таблица Кэли Z_2^3
- Подтверждён гомоморфизм psi(a XOR b) = psi(a) * psi(b) для n=3, n=6

**Используется в:** Все квантизаторы (quantizers.py), BianGuaTransform

**Статус:** ✅ Доказана и верифицирована

---

### Т2. Квантизатор-гомоморфизм при T->0

**Утверждение:** FactoredYiJingQuantizer с soft quantization сохраняет групповую структуру Z_2
в пределе T->0: Q(v1 * v2) -> Q(v1) * Q(v2) при T -> 0.

**Верификация:** `scripts/theoretical_analysis.py::theorem_2_quantizer_preserves_group()`
- MSE(Q(v1*v2), Q(v1)*Q(v2)) убывает при T: 1.0 -> 0.3 -> 0.1 -> 0.01 -> 0.001
- Hard quantization (sign): accuracy ~100% для ненулевых элементов

**Используется в:** FactoredYiJingQuantizer, temperature annealing в STE

**Статус:** ✅ Доказана и верифицирована

---

### Т3. Модулярная арифметика в Z_2^n

**Утверждение:** (a + b) mod 2^n разлагается на XOR-компоненту (линейную в Z_2) и
нелинейные переносы. Гиперкуб {-1,+1}^n нативно вычисляет XOR-часть через
покоординатное умножение.

**Верификация:** `scripts/theoretical_analysis.py::theorem_3_modular_arithmetic()`
- Ripple-carry adder через XOR+AND: 64/64 тестов пройдено для mod 8
- XOR даёт ~79% верных бит для (a+b) mod 8

**Используется в:** Обоснование гиперкубной квантизации как inductive bias

**Статус:** ✅ Верифицирована

---

### Т4. Хэмминговы расстояния через скалярное произведение

**Утверждение:** d_H(v1, v2) = 1/2 * (n - v1 . v2) для v in {-1,+1}^n.
Каждая точка имеет C(n,k) соседей на расстоянии k.
Гиперкуб содержит Reed-Muller коды как подмножества.

**Верификация:** `scripts/theoretical_analysis.py::theorem_4_hamming_structure()`
- Формула d_H проверена для n=3 (триграммы) и n=6 (гексаграммы)
- RM(1,3): 16 кодовых слов, d_min=4 — подтверждено
- Среднее расстояние = n/2 — подтверждено

**Используется в:** MultiScaleGlobalRouter (Hamming prior), HierarchicalMoE

**Статус:** ✅ Верифицирована

---

### Т5. WHT-спектральное разложение

**Утверждение:** Матрица Адамара H_n = kron^n [[1,1],[1,-1]] диагонализует свёртку на Z_2^n.
Строки H_n = характеры группы Z_2^n. Dot-product attention на гиперкубе неявно вычисляет
преобразование Уолша-Адамара.

**Верификация:** `scripts/theoretical_analysis.py::theorem_5_walsh_hadamard()`
- H_n * H_n^T = N * I — подтверждено для n=2,3
- Строки H_n = chi_a(x) = (-1)^(a.x) — подтверждено
- Свёрточная теорема: WHT(g*h) = WHT(g) . WHT(h) — ошибка < 1e-5

**Используется в:** WalshHadamardAnalyzer (extensions.py), MultiScaleGlobalRouter (WHT masks)

**Статус:** ✅ Верифицирована

---

### Т6. BianGua как стохастическая XOR-маска

**Утверждение:** BianGuaTransform z -> z * (1 - 2*sigma(w)) реализует стохастическую
XOR-маску в 6D пространстве гексаграмм. При sigma->1: z -> -z (инверсия = XOR с 1).
При sigma->0: z -> z (тождество). BianGua + HexagramAttentionPattern = полная система
управления через 6D гексаграммное пространство.

**Верификация:** `scripts/theoretical_analysis.py::theorem_6_biangua_attention()`
- change_logits -> +inf: ratio z_transformed / (-z) -> 1.0
- 6 attention паттернов слабо коррелированы (ортогональные аспекты)

**Используется в:** BianGuaTransform, GraduatedBianGuaTransform, HexagramAttentionPattern

**Статус:** ✅ Верифицирована

---

### Т7. Выразительность гиперкубной квантизации

**Утверждение:** {(-1)^(a.x) : a in Z_2^n} — полный ортонормальный базис L^2(Z_2^n).
Любая f: Z_2^n -> R представима как линейная комбинация 2^n характеров (мономы Уолша).
XOR-функция — единственный характер chi_(1,1,...,1).

**Верификация:** `scripts/theoretical_analysis.py::theorem_7_expressivity_bound()`
- 8 характеров Z_2^3 ортогональны: <chi_a, chi_b> = 8*delta_ab
- Реконструкция произвольной f: ошибка < 1e-6
- XOR = 1 моном (идеально для Z_2-геометрии)

**Используется в:** Обоснование 64 гексаграмм как полного базиса для обучения

**Статус:** ✅ Верифицирована

## Теоремы с формулировками, без полной верификации (12/19)

### Т8. Факторизация Q6 = Q3 x Q3

**Утверждение:** 64 гексаграммы = тензорное произведение двух наборов по 8 триграмм:
{-1,+1}^6 = {-1,+1}^3 x {-1,+1}^3 (нижняя триграмма x верхняя триграмма).

**Где упомянута:** CONCEPTUAL_STAGE.md, Variant3GPT (HexagramProjection)

**Используется в:** FactoredYiJingQuantizer (факторизованная квантизация), TrigramMoE

**Статус:** ⚠️ Следует из определения, формально не верифицирована в scripts/

---

### Т9. Матрёшка Q2 -> Q3 -> Q4 -> Q6

**Утверждение:** Иерархия гиперкубов Q2(4) -> Q3(8) -> Q4(16) -> Q6(64) определяет
естественную мультимасштабную квантизацию. Q4 = подкуб Q6 (первые 4 координаты).

**Где упомянута:** PSEUDORAG_YIJING_BRIDGE.md, HMoEConfig (use_q4)

**Используется в:** MultiScaleGlobalRouter (Q2->Q4->Q3->Q6), PseudoRAGProjection

**Статус:** ⚠️ Формулировка, верифицирована в validate_q4_q6.py (keyword-based)

---

### Т10. Температурный фазовый переход (Ising)

**Утверждение:** Критическая температура T_c = z*J/2 = 6*1/2 = 3.0 для mean-field Ising
на 6-мерном гиперкубе. Ниже T_c — упорядоченная фаза (доминирование одного эксперта).

**Где упомянута:** HMoEConfig.lambda_temp_reg, CONCEPTUAL_STAGE.md

**Используется в:** HierarchicalMoEFFN (topology_loss с lambda_temp_reg)

**Статус:** ⚠️ Формулировка, используется как regularization target

---

### Т11. Антиподальная регуляризация

**Утверждение:** Для каждой вершины v гиперкуба существует антиподальная -v.
Штраф на активацию антиподов: если Q(x) = v, то Q(x') != -v для близких x, x'.

**Где упомянута:** config.py (use_antipodal_reg), AntipodalQuantizer

**Используется в:** YiJingTransformerLayer (_antipodal_loss)

**Статус:** ⚠️ Эвристика, не формализована как теорема

### Т12. D4-эквивариантность

**Утверждение:** Группа диэдра D4 (симметрии квадрата) действует на подпространстве
Q6, порождая 8 эквивалентных представлений каждой гексаграммы.

**Где упомянута:** config.py (use_d4_equivariant), D4EquivariantLayer

**Используется в:** YiJingTransformerLayer (d4_layer)

**Статус:** ⚠️ Формулировка (Фомюк 2.2), не верифицирована

### Т13. Kirchhoff Current Law для маршрутизации

**Утверждение:** Сумма весов маршрутизации через все ветви графа = 1 (сохранение тока).
LCI (Loop Circuit Identity): |w_A - w_B|^2 -> 0 для балансировки ветвей figure-8.

**Где упомянута:** HMoEConfig.lambda_lci, CONCEPTUAL_STAGE.md

**Используется в:** HierarchicalMoEFFN (topology_loss Level 1)

**Статус:** ⚠️ Аналогия с физикой, используется как loss term

### Т14. Мёбиусова топология attention

**Утверждение:** Attention bias с тополовией ленты Мёбиуса создаёт
неориентируемую поверхность в пространстве attention scores, что улучшает
покрытие дальних зависимостей.

**Где упомянута:** config.py (use_mobius_bias), MobiusAttentionPattern

**Используется в:** YiJingTransformerLayer (extra_bias)

**Статус:** ⚠️ Формулировка (Беляев 6.3), не верифицирована формально

### Т15. SOLAN-76 как визуальный интерфейс Q6

**Утверждение:** 76 глифов SOLAN-76 покрывают подмножество {-1,+1}^6,
обеспечивая биективное отображение глиф <-> вершина Q6 для 76 из 256 byte-values.

**Где упомянута:** CONCEPTUAL_STAGE.md, GlyphTokenizer

**Используется в:** GlyphTokenizer._SOLAN_MAP, _build_glyph_q6_table() в model.py

**Статус:** ⚠️ Реализовано в коде, но покрытие 76/256 = 30%

### Т16. Тернарная логика {-1, 0, +1}

**Утверждение:** Расширение Z_2 до Z_3 (три состояния: инь, ян, переходное)
позволяет моделировать неопределённость (变爻). Число тернарных состояний:
3^6 = 729 (полное) или факторизованное 2*27.

**Где упомянута:** config.py (use_ternary_quantizer), TernaryQuantizer

**Используется в:** TernaryQuantizer, ArchetypalInterlingua (interlingua_use_ternary)

**Статус:** ⚠️ Реализовано в коде, теоретическое обоснование неформальное

### Т17. Flower of Life как граф attention

**Утверждение:** 7-узловой граф "Цветок Жизни" (центр + 6 соседей) определяет
adjacency matrix для Graph Attention Network, естественно связанный с
6 координатами гексаграммы.

**Где упомянута:** config.py (use_flower_gat), FlowerOfLifeGAT

**Используется в:** YiJingTransformerLayer (flower_gat), LeanYiJingGPT

**Статус:** ⚠️ Эвристическая аналогия, работает на практике (PPL 1.01 solo)

### Т18. Heisenberg uncertainty в attention temperature

**Утверждение:** Принцип неопределённости Гейзенберга delta_x * delta_p >= h/2
аналогичен trade-off между остротой (temperature) и покрытием (entropy) attention.
HeisenbergAttention динамически регулирует temperature через learned uncertainty.

**Где упомянута:** config.py (use_heisenberg_attention), HeisenbergAttention

**Используется в:** YiJingTransformerLayer, LeanYiJingGPT

**Статус:** ⚠️ Аналогия, не формализована математически

---

### Т19. PseudoRAG Q4 subset Q6

**Утверждение:** 16 PseudoRAG-архетипов вкладываются в Q6 через
phi(b1,b2,b3,b4) = (2b1-1, 2b2-1, 2b3-1, 2b4-1, 0, 0).
Метрика Хэмминга сохраняется. Каждый Q4-архетип порождает кластер из 4 гексаграмм.

**Где упомянута:** PSEUDORAG_YIJING_BRIDGE.md, models/pseudo_rag.py

**Используется в:** PseudoRAGProjection, PseudoRAGDistillationLoss, validate_q4_q6.py

**Статус:** ⚠️ Формулировка верна (тривиальна), реализована в коде
