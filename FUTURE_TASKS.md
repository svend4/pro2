## 🔴 ПРИОРИТЕТ 1 — интеграция с meta-репозиторием

### Проблема
`pro2` решает задачу эмпирически (curriculum pipeline, LCI-метрика, bench_all).
`svend4/meta` решает те же математические структуры формально и с тестами.

Верификация (2026-03-22): 10/11 claims VERIFIED, 1 PARTIAL (hexdim Q4 enumerations).

### Конкретные интеграции

**hexnet → Q6-router в HMoE**
- Сейчас в pro2: GlobalRouter с обученными весами (gate ≈ 0.489–0.494, константа)
- В meta: hexnet — детерминированная маршрутизация через Hamming-distance на Q6
- Задача: заменить GlobalRouter на hexnet.route(src, dst) как hard prior,
  оставить обученные веса только для fine-tune поверх геометрической структуры
- Ожидаемый эффект: routing не коллапсирует к константе, gate ≠ 0.5

**hexlearn → инициализация RAG**
- Сейчас: RagBuffer добавляет тексты по embedding-близости (евклидово пространство)
- В meta: hexlearn k-NN работает в геометрии Q6 (Hamming), различает все 64 архетипа
- Задача: при seed_texts заменить _get_emb → Q6 probing (проецировать embedding
  на ближайшую вершину Q6, использовать Hamming для retrieve)
- Ожидаемый эффект: RAG не вырождается в кластер одинаковых текстов

**hexsym → Aut(Q6) для routing-аугментации**
- Сейчас: 4 агента nautilus-4agent работают в кольцах 1:2:3:4 (произвольные)
- В meta: hexsym.orbits() даёт орбиты под B₆ = S₆ ⋉ (Z₂)⁶ (46080 элементов)
- Задача: вместо фиксированных колец META/ABSTRACT/DYNAMIC/CONCRETE использовать
  орбиты под Aut(Q6) — каждый агент специализируется на одной орбите
- Ожидаемый эффект: архитектурная дифференциация вместо ручной

**hexphys → Metropolis annealing в pipeline**
- Сейчас: температура фиксирована = 1.4 во всех циклах
- В meta: hexphys/hexopt — Metropolis-MCMC для корректного temperature annealing
- Задача: в pipeline.py добавить phase_temperature(cycle, max_cycle) по Metropolis
  расписанию (принимать/отвергать обновление с P = exp(−ΔE/T))
- Ожидаемый эффект: поздние циклы сходятся детерминированнее, меньше осцилляций ±0.05

**hexring → bent-функции как seed архетипы**
- Сейчас: seed_texts = 10 ручных строк + 64 hex_prompt (произвольные)
- В meta: hexring.bent() даёт bent-функции — теоретически оптимальные архетипы
  (WHT-спектр равномерный, максимальное расстояние между всеми парами)
- Задача: инициализировать RAG через bent-функции вместо hex_prompt
- Ожидаемый эффект: начальное разнообразие RAG гарантировано математически

**hexdim → Q4⊂Q6 агенты**
- Сейчас: nautilus-4agent — 4 агента по 4 кольцам (META/ABSTRACT/DYNAMIC/CONCRETE)
- В meta: hexdim находит все 15 копий Q4 внутри Q6
- Задача: создать nautilus-15agent где каждый агент специализируется на своём
  тессеракте Q4 (15 агентов × 4 вершины = 60 из 64 вершин Q6 покрыто)
- Ожидаемый эффект: более полное покрытие пространства, меньше слепых зон routing

- Тест: temperature 1.4 → 1.2 → 1.0 → 0.8 по циклам в nautilus_4agent

- Тест: 3-й проход с lr=5e-6 вместо 1e-5
- Тест: сбросить RAG между проходами (rag = RagBuffer(max_size=500) заново)

**Воспроизводимость 0.998**
