## 1. Что такое NautilusMoME

NautilusMoME — это малопараметрическая языковая модель (~3M параметров) с архитектурой **Mixture of Micro-Experts (MoME)**. Модель демонстрирует, что принципы MoE-маршрутизации работают даже при экстремально малом масштабе — задача, в которой большинство MoE-архитектур коллапсируют в одного эксперта.

### Контекст

Проект существует в рамках более широкого YiJing-Transformer — трансформерной архитектуры с индуктивным смещением на основе геометрии гиперкуба Q6 = {-1,+1}⁶. Параллельная ветка развития (v53–v61) исследует интеграцию теоретических источников (Склярова, Фомюк, Андреев, Касаткин, Герман, Беляев, SOLAN) и достигла PPL 1.24 с архитектурой AbrialeBridgeMediator.

NautilusMoME — отдельный трек: прагматичная, мультидоменная, работающая в реальных условиях модель.

## 2. Архитектура: ключевые компоненты

```
Input → core_first → Router → [6 Experts] → NautilusBridge → core_second → Output
                                                    ↓
                                        [CrossDomainAnalogy]   ← фаза 4–6b
                                                    ↓
                                          [ArchetypeLayer]     ← фаза 7–10
```

### 6 экспертов

| Эксперт  | Специализация            | Routing на домене |
|----------|--------------------------|-------------------|
| CODE     | Python, JS, React        | 0.261–0.296       |
| RECON    | Русский язык, документация | 0.559–0.655     |
| SYSTEM   | SQL, K8s, Docker, инфра  | 0.327–0.361       |
| MATH     | Формулы, алгоритмы       | 0.255             |
| HUMAN    | Общий текст, не-код      | —                 |
| INFO     | Конфигурации, YAML, README | 0.310–0.339     |

### Ключевые блоки

- **NautilusBridge** — иерархическое слияние выходов экспертов с residual gate
- **CrossDomainAnalogy** — 15 пар между 6 экспертами для кросс-доменного переноса (ProverbCondenser + AnalogyPair)
- **ArchetypeLayer** — 64-архетипный слой с тернарной квантизацией {-1, 0, +1}

### Параметры модели (текущая, v1)

| Компонент          | Параметры  |
|--------------------|------------|
| Базовая NautilusMoME | 1,822,797 |
| CrossDomainAnalogy  | 1,083,551  |
| ArchetypeLayer      | ~120,000   |
| **Итого**           | **~3,026,000** |

Гиперпараметры: `d_model=128`, `n_layers=4`, `n_heads=4`, `block_size=256`, `vocab_size=4096 (BPE)`

### Ключевые наблюдения v1

1. **Маршрутизация работает** — главный успех. MoE-routing при 3M параметрах не коллапсировал.
2. **Генерация деградирует** после 15–20 токенов (причина: `block_size=256`, `vocab=4096`).
3. **Domain mixing** — главная проблема. Код + русский + тесты в одном ответе.
4. **Routing confidence 2–3%** — эксперты активируются "мягко", нет резкой специализации.
5. **Bridge residual gate = 0.295** — эксперты влияют на 30% выхода, 70% — прямой bypass.
6. **CrossDomainAnalogy в действии** — модель вставляет русские "афоризмы" в код: это RECON + CODE синтез.

## 5. Концептуальный план v2 (Nautilus-Sharp)

**Философия v2:** *Углубить специализацию, расширить когерентность, сохранить модульность.*

### Уровень 1: Исправления инференса (без переобучения)

**Effort: 1–2 дня | Impact: Средний**

#### 1a. Domain-Locked Generation

**Проблема:** Генерация смешивает домены после 15 токенов.

**Решение:**
```
1. Определить доминирующего эксперта на первых 5 токенах
2. Усилить его вес (× 1.5–2.0)
3. Подавить конкурирующих экспертов (× 0.3–0.5)
4. Переключение только при высокой энтропии (> 0.6)
```

#### 1b. Coherence Penalty в генерации

**Проблема:** Нет штрафа за резкое переключение домена.

**Решение:** Отслеживать routing history (последние 10 шагов). Штрафовать токены с резким routing shift: `shift = KL(routing_now, routing_avg_last_10)`.

#### 1c. Mirostat-подобный sampling

**Проблема:** Стандартный top-p/top-k недостаточен для когерентности.

**Решение:** Target surprise level τ = 5.0 бит. Адаптивная температура для стабилизации генерации.

---

### Уровень 2: Новые фазы обучения

**Effort: 3–5 дней | Impact: Высокий**

#### Phase 11: Sharper Expert Routing (1000 шагов)

**Цель:** Routing confidence 2–3% → 15–25%.

```python
# Изменения:
# 1. Температура маршрутизации: 1.0 → 0.3 (резче выбор)
# 2. Contrastive routing loss:
#    loss += max(0, margin - routing[E_D] + max(routing[E_other]))
# 3. Expert dropout: случайное отключение top-1 эксперта
#    → заставляет top-2 специализироваться

LR = "5e-5 (только router + bridge)"
```

#### Phase 12: Coherence Training (800 шагов)

**Цель:** Стабильная генерация на 64+ токенов.

```
Self-Consistency Training:
1. Генерировать 64 токена (teacher-forced)
2. Разбить на окна [0:32] и [32:64]
3. Routing первого окна → routing target для второго
4. Loss: MSE(routing_window2, routing_window1) × α
```

#### Phase 13: Russian Language Specialist (1000 шагов)

**Цель:** PPL на русском 64.2 → < 30.

```
1. Больше русских данных из репозиториев
2. Дообучить RECON-эксперт на русском
3. BPE-субтокенизация для кириллицы
4. Cross-attention между RECON и CODE
   (для bilingual текста: русские комментарии в коде)

LR = "1e-4 (только RECON + bridge)"
```

### Уровень 3: Архитектурные изменения

**Effort: 1–2 недели | Impact: Трансформационный**

#### 3a. Scale-Up: 3M → 10–15M параметров

| Параметр | v1 | v2 |
|----------|----|----|
| d_model | 128 | **256** |
| n_layers | 4 | **6** (3+3 split) |
| n_heads | 4 | **8** |
| d_expert | 128 | **256** |
| block_size | 256 | **512** |
| vocab_size | 4096 | **8192** |
| Ожидаемые параметры | 3M | **~12M** |
| Ожидаемый PPL | ~18 | **~8–12** |

#### 3b. Domain-Aware NautilusBridge

**Проблема:** Bridge смешивает экспертов механически.

```
Текущий bridge:
  expert_outputs → pairwise_merge → global_merge + residual

Domain-Aware Bridge:
  Query = core_hidden
  Keys = Values = expert_outputs
  Output = Attention(Q, K, V) — контекстно-зависимое смешивание
```

#### 3c. Expert-Specific Memory (KV-Cache per Expert)

```
CODE  → KV-cache 64 слота  (последние 64 строки кода)
RECON → KV-cache 64 слота  (последние 64 русских слова)
MATH  → KV-cache 32 слота  (числа и формулы)

Эффективный контекст: 256 + 6×64 = 640 токенов
```

#### 3d. Lightweight Instruction Head

```
Input:  первые 32 токена промпта
Output: {CONTINUE, ANSWER, CODE_COMPLETE, TRANSLATE}

Эффект: routing hint (не full instruction tuning)
Требует: ~500 размеченных пар (генерируются автоматически)
```

## Вывод

NautilusMoME v1 — это успешная демонстрация трёх ключевых принципов:

1. **MoE-маршрутизация работает при 3M параметрах** — главный результат, опровергающий тезис о минимальном масштабе для MoE.
2. **CrossDomainAnalogy порождает неожиданное поведение** — вставка русских "афоризмов" в код (RECON + CODE = "кодовая философия") — это именно та кросс-доменная креативность, для которой проектировалась архитектура.
3. **Честная калиброванность** — PPL gap 12.6x показывает, что модель знает свои границы.

v2 (Nautilus-Sharp) — это переход от "работает" к "работает хорошо": резкая специализация экспертов, когерентная генерация, расширенный масштаб.
