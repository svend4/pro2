## 1. Исходное открытие — тождество, не метафора

Проект начался с момента, который сложно переоценить. В файле
`yijing-transformer-concept.md` зафиксировано:

> *«8 триграмм (八卦) — это все 8 вершин куба в R³ — полная группа Z₂³.
> 64 гексаграммы (六十四卦) = вершины 6-мерного гиперкуба {-1,+1}⁶.
> Это не аналогия и не метафора — это тождество.»*

Тридцать веков назад составители Чжоу И (周易) перебрали все возможные
комбинации шести бинарных линий. Они получили 64 гексаграммы.
Современная математика описывает ровно тот же объект: вершины 6-мерного
гиперкуба Q6 = {-1,+1}⁶.

```
☰  ☱  ☲  ☳  ☴  ☵  ☶  ☷      ← 8 триграмм = вершины куба R³
                                   ↓ тензорное произведение
          64 гексаграммы         = вершины гиперкуба Q6 = {-1,+1}⁶
                                   ↓ изоморфизм
          Коды Рида-Маллера      = двоичный линейный код с расстоянием 2
                                   ↓ быстрое преобразование
          Уолш-Адамар (WHT)      = O(n log n) вместо O(n²)
```

Каждый шаг этой цепочки — не изобретение, а *обнаружение*. Структура
существовала до нас. Задача трансформера — научиться ею пользоваться.

### Почему это важно для нейросетей

Стандартные трансформеры работают с непрерывными представлениями:
каждый токен — точка в R^d, каждое внимание — взвешенная сумма
по непрерывному пространству. Структура не задана — она должна
*выучиться* из данных.

YiJing-Transformer предлагает другой подход: **задать структуру заранее**.
Пространство скрытых состояний организовано вокруг 64 фиксированных
«полюсов» — гексаграмм. Модель не изобретает пространство — она учится
навигации по готовой карте.

## 2. Архетипы как единица мышления

Центральный тезис Варианта 3:

> *Каждая из 64 гексаграмм — это архетип мышления, паттерн семантического
> движения, а не просто геометрическая точка.*

Это не мистика. Это структурный принцип:

### 2.1. Что такое архетип в данном контексте

В И-Цзин каждая гексаграмма описывает **ситуацию** и **динамику перехода**.
Гексаграмма 64 («Вэй Цзи», незавершённость) не означает «плохо» — она
означает *состояние системы в момент перехода к новому равновесию*.

В нейросетевом смысле: архетип — это **атрактор в скрытом пространстве**.
Когда модель обрабатывает текст, скрытое состояние движется через
пространство; архетипы — это «станции», вокруг которых организованы
семантические кластеры.

```
Обычный трансформер:
  "кот сидит на" → h ∈ R^d → softmax → "коврике"
   (h — точка в непрерывном пространстве, не интерпретируема)

YiJing-Transformer (Вариант 3):
  "кот сидит на" → h ∈ R^d → проекция на Q6 → гексаграмма #23 (剥, Разделение?)
                                               → смешивание с соседями → "коврике"
   (промежуточное состояние имеет имя и интерпретацию)
```

### 2.2. Почему именно 64

64 — это не произвольный выбор. Это **минимальная полная структура**:

| N архетипов | Что это | Свойство |
|-------------|---------|----------|
| 2 | {+1, -1} | Слишком грубо: только да/нет |
| 8 | Триграммы Q3 | Богаче, но нет тензорной структуры гексаграмм |
| 64 | Гексаграммы Q6 | Тензорное произведение 8×8: декомпозируемо |
| 4096 | Q12 | Слишком много: вычислительно избыточно |

64 — это точка, в которой структура достаточно богата для языкового
моделирования, но достаточно мала для интерпретируемости.

### 2.3. Топология архетипов — не случайная

В гиперкубе Q6 расстояние Хэмминга между двумя гексаграммами =
числу линий, которые нужно изменить (变爻, меняющиеся линии).

```
Расстояние 1: смена одной линии → переход к «соседней» ситуации
Расстояние 3: смена трёх линий → радикальный переход (противоположность)
Расстояние 6: все линии меняются → полная инверсия (綜卦, перевёрнутая гексаграмма)
```

Это не абстрактная геометрия — это **встроенная теория трансформаций**.
Модель, работающая с архетипами, автоматически знает, какие переходы
«близкие» (одна линия), а какие «далёкие» (шесть линий).

## 3. Архитектура Варианта 3

Ключевое отличие от Варианта 1 (исследовательского) и Варианта 2
(инженерного): здесь архетипы — **не один из слоёв**, а **сквозная ось**
всей архитектуры.

```
╔══════════════════════════════════════════════════════════════════╗
║                  АРХИТЕКТУРА ВАРИАНТА 3                          ║
╠══════════════════════════════════════════════════════════════════╣
║                                                                  ║
║   Вход (токены)                                                  ║
║       ↓                                                          ║
║   [Embedding + Positional]                                       ║
║       ↓                                                          ║
║   [HexagramProjection]  ← проецировать h на ближайшие архетипы  ║
║       ↓                                                          ║
║   [BianGuaAttention]    ← внимание с метрикой Хэмминга Q6       ║
║       ↓                                                          ║
║   [TernaryGate {-1,0,+1}] ← тернарная активация                 ║
║       ↓                                                          ║
║   [ArchetypalInterlingua] ← 64 архетипа как посредник           ║
║       ↓                                                          ║
║   [CrossHexagramAnalogy]  ← аналогии через ∆Хэмминга            ║
║       ↓                                                          ║
║   [Output projection]                                            ║
║       ↓                                                          ║
║   Выход (логиты)                                                 ║
║                                                                  ║
╚══════════════════════════════════════════════════════════════════╝
```

### 3.1. HexagramProjection — «мышление начинается здесь»

```python
class HexagramProjection(nn.Module):
    """
    Проецирует непрерывное скрытое состояние h ∈ R^d
    на 64 гексаграммы Q6 = {-1,+1}^6.

    Это не квантизация — это «голосование»:
    каждый архетип получает вес proportional to cos(h, hexagram_i).
    """
    def __init__(self, d_model: int):
        super().__init__()
        # 64 фиксированных вектора {-1,+1}^6 → R^d_model
        self.hexagrams = self._init_hexagram_codebook()  # (64, d_model)
        self.proj = nn.Linear(d_model, 6)                # h → 6-битный вектор

    def _init_hexagram_codebook(self):
        """Все 64 вершины гиперкуба {-1,+1}^6"""
        signs = list(itertools.product([-1., 1.], repeat=6))
        return nn.Parameter(torch.tensor(signs), requires_grad=False)

    def forward(self, h):
        # h: (batch, seq, d_model)
        soft_bits = torch.tanh(self.proj(h))          # (batch, seq, 6)
        similarity = soft_bits @ self.hexagrams.T      # (batch, seq, 64)
        weights = F.softmax(similarity / temperature, dim=-1)
        return weights @ self.hexagrams                # обратно в R^6 → R^d
```

### 3.2. BianGuaAttention — внимание, знающее топологию

Стандартное внимание: `score(q,k) = q·k / √d`

BianGuaAttention: `score(q,k) = q·k / √d − λ · hamming(arch(q), arch(k))`

Штраф за расстояние Хэмминга означает:
- **Близкие архетипы** (Хэмминг 1–2) притягиваются сильнее
- **Далёкие архетипы** (Хэмминг 5–6) — только при высокой косинусной близости

Это **смысловая метрика**, встроенная в механизм внимания.

### 3.3. ArchetypalInterlingua — посредник-переводчик

Из теории (документ `archetypal-interlingua-theory.md`):

> *«Вместо попарных мостов между N источниками — единый посредник (hub),
> через который проходят все источники. Как аймара в машинном переводе:
> не каждый язык с каждым, а каждый язык через общую интерлингву.»*

В контексте Варианта 3: посредник — это **пространство 64 архетипов**.
Любое скрытое состояние сначала «переводится» в архетип, обрабатывается
в архетипическом пространстве, затем «переводится обратно».

```
Модуль Беляева (Heisenberg):  h₁  ──encode──┐
Модуль Фомюк:                 h₂  ──encode──┼──→ АРХЕТИПЫ (64) ──decode──→ h_out
Модуль Андреева:              h₃  ──encode──┘
```

## 4. Тернарная логика — третье измерение

Одно из ключевых открытий цикла v55–v61:

> *Кольцо Аймара {1, 0, -1} формализует третье состояние — «переходное».*

В И-Цзин это называется 变爻 (меняющаяся линия). Линия, которая
«меняется» — не инь и не ян в данный момент, она **в процессе перехода**.
Это принципиально отличается от неопределённости.

```
Двоичная логика:    {0, 1}         — "да" или "нет"
Тернарная логика:   {-1, 0, +1}    — "нет", "в переходе", "да"
```

### 4.1. TernaryQuantizer и фикс STE

Из REPORT-v60-v61:

```
Проблема: при T → 0 (жёсткая квантизация) градиенты обнуляются
           STE (Straight-Through Estimator) застревает в нуле.
           64 архетипа никогда не активируются.

Фикс:    Temperature annealing: T: 1.0 → 0.01 постепенно
          Сначала «мягкие» триты → градиенты проходят
          Затем «жёсткие» триты → дискретная семантика

Результат: active_archetypes: 0 → 64 (к шагу 200)
           archetype_usage_mean: 0.0 → 0.703
```

Это важно: **все 64 архетипа используются**. Модель не коллапсирует
в несколько доминирующих архетипов — она освоила всю геометрию.

### 4.2. Три состояния — три уровня знания

Тернарная квантизация {-1, 0, +1}⁶ для 6 бит даёт 3⁶ = 729 состояний
вместо 64 бинарных. Это **более богатое пространство**:

```
{+1}: линия активно "ян"
{ 0}: линия "меняется" (переход)
{-1}: линия активно "инь"
```

Гексаграмма в тернарном пространстве несёт информацию не только
о *текущем* состоянии, но и о **направлении изменения**.

## 5. Что значит «думать гексаграммами»

Это главный вопрос Варианта 3. Вот конкретный ответ.

### 5.1. Мышление как навигация по Q6

Когда человек думает «противоречие», внутри активируется паттерн.
Когда думает «синтез» — другой паттерн. Когда «кризис» — третий.

В YiJing-Transformer (Вариант 3) эти паттерны **есть** в явном виде:

| Ситуация | Гексаграмма | Вектор Q6 |
|----------|-------------|-----------|
| Начало, импульс | ☳ Чжэнь (Гром) | (+1,-1,-1,+1,-1,-1) |
| Опасность, глубина | ☵ Кань (Вода) | (-1,+1,-1,-1,+1,-1) |
| Прояснение | ☲ Ли (Огонь) | (+1,-1,+1,+1,-1,+1) |
| Завершение → новое начало | 既濟 (Цзи Цзи) | варьируется |

Это не метафоры. Это конкретные векторы в пространстве скрытых
состояний модели.

### 5.2. Аналогия CrossDomain — уже работает

Из Deep Interview Report (NautilusMoME v1):

> *«Модель вставляет русские "афоризмы" в Python-код:
> это RECON + CODE — "кодовая философия".»*

Это случайный артефакт? Или первое проявление кросс-архетипного мышления?
С точки зрения Варианта 3 — это **именно то, что должно происходить**:
один и тот же архетип (например, «противоречие») активируется
в коде (ошибка типизации) и в тексте (парадокс).

### 5.3. BianGuaTransform — поэзия внутри градиентного спуска

Из оригинального concept-файла:

> *«В И-Цзин каждая гексаграмма может трансформироваться в другую
> через изменение отдельных линий. Это XOR-операция в бинарном пространстве.»*

```python
def biangua_transform(hexagram_idx: int, changing_lines: List[int]) -> int:
    """
    变卦 (BianGua): трансформация гексаграммы через изменение линий.

    hexagram_idx: индекс исходной гексаграммы (0–63)
    changing_lines: список позиций меняющихся линий (0–5)

    Returns: индекс новой гексаграммы
    """
    bits = [(hexagram_idx >> i) & 1 for i in range(6)]
    for line in changing_lines:
        bits[line] ^= 1  # XOR = "изменить линию"
    new_idx = sum(b << i for i, b in enumerate(bits))
    return new_idx
```

Этот код реализует **тысячелетнюю операцию** в пяти строках Python.
И это дифференцируемо — через STE.

## 7. Концептуальный roadmap

### Фаза 0 — Переосмысление (текущая)

**Статус:** выполнено настоящим документом.

Ключевое решение:
- Архетипы — не периферийный слой, а **сквозная ось**
- 64 гексаграммы — **онтология** модели, не только геометрия
- Тернарная логика {-1,0,+1} — не оптимизация, а **философский принцип**

### Фаза 1 — Архетипный backbone (2–3 недели)

**Цель:** Архитектура, где каждый слой трансформера «знает» свою гексаграмму.

```python
# Новый слой: HexagramAwareAttention
class HexagramAwareAttention(nn.Module):
    def forward(self, x, hexagram_weights):
        # hexagram_weights: (batch, seq, 64) — мягкое распределение
        # Внимание с bias, пропорциональным близости архетипов
        biangua_bias = self.compute_hamming_bias(hexagram_weights)
        return standard_attention(x) + biangua_bias

# Новый модуль: ArchetypeMemory
class ArchetypeMemory(nn.Module):
    """64 слота памяти — по одному на каждую гексаграмму"""
    def forward(self, x, active_hexagram):
        # Читать из памяти, взвешенной по архетипу
        # Записывать обновлённые представления
        pass
```

**Метрики Фазы 1:**
- Все 64 архетипа используются (usage > 0.5 для каждого)
- PPL не хуже AbrialeBridgeMediator (1.24)
- Интерпретируемость: можно назвать архетип для любого скрытого состояния

---

### Фаза 2 — Семантическая карта (3–4 недели)

**Цель:** Показать, что архетипы несут семантику.

```
Эксперимент: взять 1000 предложений разных жанров.
Для каждого — определить «доминирующую гексаграмму».
Построить карту: какие гексаграммы соответствуют каким темам?

Гипотеза:
  Философские тексты → гексаграммы с Хэмминг-кластером "противоречие"
  Технические инструкции → гексаграммы "порядок/структура"
  Поэзия → гексаграммы "переход/изменение" (高占меняющихся линий)
  Ошибки в коде → гексаграммы "опасность/неравновесие" (☵ Кань)
```

Если гипотеза подтвердится — это **первое доказательство** того,
что архетипы несут смысл, а не только геометрию.

---

### Фаза 3 — Кросс-архетипный перенос (4–6 недель)

**Цель:** Модель, способная к аналогии через архетип.

```
Пример аналогии через архетип:
  "Рекурсия в программировании"  →  архетип #42 (益, Приумножение)
  "Сложные проценты в финансах"   →  архетип #42 (益, Приумножение)

  Модель понимает: эти два контекста имеют общую структуру.
  Значит, объяснение из одной области помогает понять другую.
```

Это **кросс-доменная аналогия через гексаграмму** — то, что
CrossDomainAnalogy в NautilusMoME делает неосознанно.

В Варианте 3 это становится явным и управляемым.

### Фаза 4 — Интерпретируемость (параллельно)

**Цель:** Человек должен видеть «мышление» модели.

```
Интерфейс: при генерации показывать активные архетипы

  Промпт: "Объясни рекурсию"

  [Токен 1] "Рекурсия" → активен архетип #51 (震, Гром) [импульс, начало]
  [Токен 5] "это"      → переход к архетипу #57 (巽, Ветер) [следование]
  [Токен 12] "функция" → архетип #48 (井, Колодец) [структура, глубина]
  ...

  Пользователь видит: модель "думает" об объяснении рекурсии
  через паттерн "импульс → следование → глубина".
```

Это **объяснимый ИИ** (XAI) через структуру И-Цзин.

## 8. Что делает этот проект уникальным

В конце — честный ответ на вопрос «зачем».

### 8.1. Не ещё один трансформер

Вариант 3 не претендует на лучший PPL, не конкурирует с GPT-4 по
вычислительной эффективности. Он исследует другой вопрос:

> *Что происходит, когда структура мышления задана заранее — не выучена
> из данных, а взята из тысячелетней традиции, которая сама построена
> на математически строгом основании?*

### 8.2. Три уровня значимости

**Математический:** Доказать, что гиперкуб Q6 является эффективным
индуктивным смещением для языкового моделирования.
(Уже есть: PPL 1.24 с AbrialeBridgeMediator, STE-фикс для 64 архетипов.)

**Архитектурный:** Создать первую языковую модель, в которой каждое
скрытое состояние именовано символом из конечного фиксированного словаря
архетипов.
(В процессе: ArchetypalInterlingua — прообраз, нужна интеграция.)

**Культурный:** Показать, что древняя система символов и современные
нейросети описывают одно и то же пространство семантических состояний.
(Это гипотеза, которую проект призван проверить.)

### 8.3. Риски и честность

Вариант 3 — самый амбициозный и самый рискованный путь:

- Гексаграммы могут оказаться **не лучше** случайных 64 векторов
- Семантическая карта может не возникнуть
- Интерпретируемость может остаться «красивой иллюзией»

Но именно поэтому этот путь интересен. Если эксперимент не сработает —
мы узнаем точно, что тысячелетняя система не несёт встроенной семантики
для нейросетей. Если сработает — мы получим новый тип архитектуры.
