# И-Цзин как геометрическая структура для нейросетей: исследование

## 1. Математическая формализация Книги Перемен

### 1.1. Триграммы — вершины 3-мерного гиперкуба

8 триграмм (八卦) — это все комбинации 3 бинарных линий (инь ⚋ = -1, ян ⚊ = +1):

| # | Триграмма | Бинарный | Вектор в {-1,+1}³ | Название |
|---|-----------|----------|--------------------|----------|
| 0 | ☰ | 111 | (+1, +1, +1) | Цянь (Небо) |
| 1 | ☱ | 110 | (+1, +1, -1) | Дуй (Озеро) |
| 2 | ☲ | 101 | (+1, -1, +1) | Ли (Огонь) |
| 3 | ☳ | 100 | (+1, -1, -1) | Чжэнь (Гром) |
| 4 | ☴ | 011 | (-1, +1, +1) | Сюнь (Ветер) |
| 5 | ☵ | 010 | (-1, +1, -1) | Кань (Вода) |
| 6 | ☶ | 001 | (-1, -1, +1) | Гэнь (Гора) |
| 7 | ☷ | 000 | (-1, -1, -1) | Кунь (Земля) |

Это **все 8 вершин куба** в R³ — полная группа Z₂³.

### 1.2. Гексаграммы — вершины 6-мерного гиперкуба

64 гексаграммы (六十四卦) = все пары триграмм (верхняя ⊕ нижняя):

- Каждая гексаграмма — **6-битный вектор** в {-1, +1}⁶
- 64 = 2⁶ — это **все вершины 6-мерного гиперкуба**
- Тензорная структура: гексаграмма = (верхняя триграмма) ⊗ (нижняя триграмма)

```
Гексаграмма #42 (益, Приумножение):
  верхняя: ☴ Сюнь (011) = (-1, +1, +1)
  нижняя:  ☳ Чжэнь (100) = (+1, -1, -1)
  полный вектор: (-1, +1, +1, +1, -1, -1) ∈ {-1,+1}⁶
```

## 2. Сравнение: E8 vs И-Цзин (гиперкуб) vs случайные точки

| Свойство | E8 | И-Цзин (6D гиперкуб) | Случайные точки |
|----------|----|-----------------------|-----------------|
| Размерность | 8 | 6 | Любая |
| Число точек | 240 | 64 | Произвольное |
| Группа симметрий | Исключительная группа Ли | Z₂⁶ (абелева) | Нет |
| Норма всех точек | √2 | √6 | Варьируется |
| Расстояние между соседями | √2 | 2√2 (Хэмминг=2) | Варьируется |
| Упаковка сфер | Оптимальная в R⁸ | НЕ оптимальная в R⁶ | Нет гарантий |
| Алгебраическая структура | Корневая система | Булева алгебра / код Рида-Маллера | Нет |
| Вычислительная сложность | O(240·d) | O(64·d) — **в 3.75x быстрее** | O(n·d) |
| Тензорное разложение | Нет простого | **Есть: 8×8** | Нет |

### Ключевое преимущество И-Цзин: тензорная факторизация

E8 — монолитная структура из 240 точек. И-Цзин допускает **факторизацию**:

```
Квантизация к 64 гексаграммам = Квантизация к 8 триграммам (верх) × Квантизация к 8 триграммам (низ)
```

Это означает: вместо softmax по 64 точкам можно сделать **два softmax по 8 точкам**, что экспоненциально дешевле и лучше масштабируется.

## 3. Архитектура: YiJing-Transformer (концепт)

### 3.1. Кодбук гексаграмм

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import itertools

def generate_trigrams():
    """8 триграмм — вершины куба {-1, +1}³"""
    signs = [-1.0, 1.0]
    trigrams = torch.tensor(list(itertools.product(signs, repeat=3)), dtype=torch.float32)
    return trigrams  # shape: (8, 3)

def generate_hexagrams():
    """64 гексаграммы — вершины гиперкуба {-1, +1}⁶"""
    signs = [-1.0, 1.0]
    hexagrams = torch.tensor(list(itertools.product(signs, repeat=6)), dtype=torch.float32)
    return hexagrams  # shape: (64, 6)

def generate_hexagrams_factored():
    """64 гексаграммы как тензорное произведение 8 × 8 триграмм"""
    trigrams = generate_trigrams()  # (8, 3)
    upper = trigrams.unsqueeze(1).expand(8, 8, 3).reshape(64, 3)
    lower = trigrams.unsqueeze(0).expand(8, 8, 3).reshape(64, 3)
    hexagrams = torch.cat([upper, lower], dim=1)  # (64, 6)
    return hexagrams, trigrams
```

### 3.2. Факторизованный квантизатор (ключевая инновация)

```python
class YiJingQuantizer(nn.Module):
    """
    Факторизованная квантизация: проецируем в 6D,
    квантуем отдельно верхнюю (3D) и нижнюю (3D) триграммы,
    затем собираем обратно.

    Сложность: O(2 × 8 × d) вместо O(64 × d) у наивного подхода.
    """
    def __init__(self, temp=0.3):
        super().__init__()
        self.temp = temp
        trigrams = generate_trigrams()  # (8, 3)
        self.register_buffer('trigrams', trigrams)

    def forward(self, x):
        # x shape: (..., 6)
        upper, lower = x[..., :3], x[..., 3:]  # разделяем на триграммы

        # Мягкая квантизация каждой триграммы отдельно
        upper_q = self._soft_quantize(upper)
        lower_q = self._soft_quantize(lower)

        # Собираем гексаграмму
        quantized = torch.cat([upper_q, lower_q], dim=-1)

        # Straight-through estimator
        return x + (quantized - x).detach()

    def _soft_quantize(self, z):
        # z shape: (..., 3), trigrams shape: (8, 3)
        # Расстояния до 8 вершин куба
        dists = torch.cdist(z.unsqueeze(-2), self.trigrams.unsqueeze(0)).squeeze(-2)
        weights = F.softmax(-dists / self.temp, dim=-1)
        return weights @ self.trigrams

    def hard_quantize(self, x):
        """Жёсткая квантизация: каждую координату → ±1"""
        return torch.sign(x)  # Для гиперкуба это тривиально!
```

**Обратите внимание:** `hard_quantize` для гиперкуба — это просто `sign(x)`! Для E8 нужен полный поиск ближайшего из 240 соседей. Это огромное вычислительное преимущество.

### 3.3. Attention с геометрическим bias на основе триграмм

```python
class YiJingAttention(nn.Module):
    """
    Attention с bias из структуры И-Цзин.

    Идея: каждая голова attention «смотрит» вдоль оси одной из 8 триграмм.
    Триграммы ортогональнее друг другу, чем первые 8 корней E8.
    """
    def __init__(self, cfg):
        super().__init__()
        assert cfg.d_model % cfg.n_heads == 0
        self.n_heads = cfg.n_heads
        self.head_dim = cfg.d_model // cfg.n_heads
        self.scale = 1.0 / (self.head_dim ** 0.5)

        self.qkv = nn.Linear(cfg.d_model, 3 * cfg.d_model, bias=False)
        self.out = nn.Linear(cfg.d_model, cfg.d_model, bias=False)
        self.dropout = nn.Dropout(cfg.dropout)

        # 8 триграмм как направления для 8 голов
        trigrams = generate_trigrams()  # (8, 3)
        # Нормализуем: ||trigram|| = √3 → 1
        trigrams_norm = F.normalize(trigrams, p=2, dim=1)
        self.register_buffer('head_dirs', trigrams_norm[:cfg.n_heads])  # (n_heads, 3)

        # Обучаемые масштабы (начинаем с 0, как в Lila-E8)
        self.head_scales = nn.Parameter(torch.zeros(cfg.n_heads))

        self.register_buffer("causal_mask",
            torch.tril(torch.ones(1, 1, cfg.block_size, cfg.block_size)))

    def forward(self, x):
        B, T, C = x.shape
        qkv = self.qkv(x).reshape(B, T, 3, self.n_heads, self.head_dim)
        qkv = qkv.permute(2, 0, 3, 1, 4)
        q, k, v = qkv.unbind(0)

        # Стандартный attention
        scores = (q @ k.transpose(-2, -1)) * self.scale
        scores = scores.masked_fill(self.causal_mask[:,:,:T,:T] == 0, float('-inf'))

        # Геометрический bias из триграмм
        if self.head_dim >= 3:
            q3 = q[..., :3]  # первые 3 измерения
            k3 = k[..., :3]
            q_proj = torch.einsum('bhtd,hd->bht', q3, self.head_dirs)
            k_proj = torch.einsum('bhtd,hd->bht', k3, self.head_dirs)
            geo_bias = q_proj.unsqueeze(-1) * k_proj.unsqueeze(-2)
            scores = scores + self.head_scales.view(1, -1, 1, 1) * geo_bias

        attn = F.softmax(scores, dim=-1)
        attn = self.dropout(attn)
        out = (attn @ v).transpose(1, 2).reshape(B, T, C)
        return self.out(out)
```

### 3.4. Трансформация (变卦) как операция в латентном пространстве

В И-Цзин каждая гексаграмма может трансформироваться в другую через изменение отдельных линий. Это **XOR-операция** в бинарном пространстве:

```python
class BianGuaTransform(nn.Module):
    """
    变卦 (Трансформация гексаграмм).

    Моделирует переход между состояниями через переключение линий.
    В бинарном пространстве: новая_гексаграмма = старая XOR маска.
    В непрерывном: обучаемое покоординатное отражение.
    """
    def __init__(self, d_model):
        super().__init__()
        self.proj_to_6d = nn.Linear(d_model, 6, bias=False)
        self.proj_from_6d = nn.Linear(6, d_model, bias=False)

        # Вероятности "перемены" каждой линии (обучаемые)
        self.change_logits = nn.Parameter(torch.zeros(6))
        self.scale = nn.Parameter(torch.tensor(0.01))

    def forward(self, x):
        z = self.proj_to_6d(x)                    # (B, T, 6)

        # Мягкая маска перемен: вероятность переключения каждой линии
        change_prob = torch.sigmoid(self.change_logits)  # (6,)

        # Перемена: отражение координаты = умножение на -1
        # Мягкий вариант: интерполяция между z и -z
        z_changed = z * (1 - 2 * change_prob)     # при prob=1: z→-z, при prob=0: z→z

        out = self.proj_from_6d(z_changed)
        return x + self.scale * out
```

### 3.5. Полная архитектура слоя

```python
class YiJingTransformerLayer(nn.Module):
    def __init__(self, cfg):
        super().__init__()
        self.ln_attn = nn.LayerNorm(cfg.d_model)
        self.attn = YiJingAttention(cfg)

        # 六爻 (Шесть линий) — E8-аналог, но в 6D
        self.ln_yijing = nn.LayerNorm(cfg.d_model)
        self.to_6d = nn.Linear(cfg.d_model, 6, bias=False)
        self.from_6d = nn.Linear(6, cfg.d_model, bias=False)
        self.quantizer = YiJingQuantizer(temp=0.3)
        self.yijing_scale = nn.Parameter(torch.tensor(0.01))

        # 变卦 (Трансформация)
        self.bian_gua = BianGuaTransform(cfg.d_model)

        # FFN
        self.ln_ffn = nn.LayerNorm(cfg.d_model)
        self.ffn = nn.Sequential(
            nn.Linear(cfg.d_model, 4 * cfg.d_model),
            nn.GELU(),
            nn.Linear(4 * cfg.d_model, cfg.d_model),
            nn.Dropout(cfg.dropout)
        )

    def forward(self, x):
        # Attention с триграммным bias
        x = x + self.attn(self.ln_attn(x))

        # Квантизация к гексаграммам (6D гиперкуб)
        h = self.ln_yijing(x)
        z6 = self.to_6d(h)
        z6_q = self.quantizer(z6)
        x = x + self.yijing_scale * self.from_6d(z6_q)

        # Трансформация гексаграмм (变卦)
        x = self.bian_gua(x)

        # FFN
        x = x + self.ffn(self.ln_ffn(x))
        return x
```

## 4. Теоретические преимущества подхода И-Цзин

### 4.1. Вычислительная эффективность

```
E8 квантизация:     softmax(240) = O(240 × batch × seq)
И-Цзин (наивная):   softmax(64)  = O(64 × batch × seq)
И-Цзин (факторная): 2 × softmax(8) = O(16 × batch × seq)  ← в 15x быстрее E8!
```

### 4.2. Теория кодирования

Вершины гиперкуба {-1,+1}⁶ — это **код Рида-Маллера первого порядка**. Такие коды:
- Имеют максимальное минимальное расстояние для данного числа кодовых слов
- Оптимальны для передачи информации по шумному каналу (теорема Шеннона)
- Допускают быстрое декодирование через преобразование Уолша-Адамара (O(n log n))

### 4.3. Бинарная структура и язык

Аргумент в пользу гиперкуба для NLP:
- Язык содержит много бинарных оппозиций (горячо/холодно, большой/маленький, да/нет)
- Гексаграммы И-Цзин моделируют именно такие оппозиции
- Каждая координата = ось семантической оппозиции (обучаемая)
- Тензорная структура (верх × низ) может моделировать **композиционность** семантики

### 4.4. Связь с преобразованием Уолша-Адамара

Для квантизации к вершинам гиперкуба можно использовать **быстрое преобразование Уолша-Адамара** (WHT):

```python
def walsh_hadamard_quantize(x, dim=6):
    """
    Быстрая квантизация через WHT.
    Сложность: O(d × log d) вместо O(d × 2^d)
    """
    # WHT находит ближайшую вершину гиперкуба за O(d log d)
    return torch.sign(x)  # Для {-1,+1}^d это просто sign!
```

Для E8 не существует аналогичного быстрого алгоритма.

## 5. Потенциальные проблемы

| Проблема | Описание | Возможное решение |
|----------|----------|-------------------|
| Мало точек | 64 vs 240 у E8 — меньше «разрешение» | Использовать вложенные гиперкубы (6D + 8D + ...) |
| Меньшая размерность | 6D vs 8D — меньше ёмкость | Компенсировать глубиной (больше слоёв) |
| Тривиальная симметрия | Z₂⁶ проще E8 — меньше математической «магии» | Может быть преимуществом: проще анализировать |
| Нет оптимальной упаковки | Гиперкуб НЕ оптимален в R⁶ | Для дискретной квантизации это не критично |
| Gradient flow через sign | sign(x) имеет нулевой градиент | STE + temperature softmax (как у E8) |

## 6. Расширения: от 64 к 4096

### 6.1. Иерархические гексаграммы

И-Цзин допускает **рекурсивную вложенность**:

```
Уровень 0: 2 состояния (инь/ян)         — {-1, +1}¹
Уровень 1: 8 триграмм                    — {-1, +1}³
Уровень 2: 64 гексаграммы                — {-1, +1}⁶
Уровень 3: 4096 «двойных гексаграмм»     — {-1, +1}¹²
```

На уровне 3 получаем 4096 точек в 12D — сопоставимо с E8 по мощности, но с полной тензорной факторизацией: `8 × 8 × 8 × 8 = 4096`, квантизация через `4 × softmax(8)`.

### 6.2. Взвешенные линии (老阴, 老阳, 少阴, 少阳)

В традиционном гадании И-Цзин каждая линия имеет 4 состояния, а не 2:
- 老阳 (старый ян) = +1, изменяющийся
- 少阳 (молодой ян) = +0.5, стабильный
- 少阴 (молодая инь) = -0.5, стабильная
- 老阴 (старая инь) = -1, изменяющаяся

Это даёт `4⁶ = 4096` состояний в непрерывном пространстве {-1, -0.5, +0.5, +1}⁶ — **более плотный кодбук** без увеличения размерности.

## 8. Заключение

Подход на основе И-Цзин **математически обоснован и потенциально превосходит E8** для задачи квантизации латентного пространства по нескольким ключевым метрикам:

1. **Тензорная факторизация** (8×8 вместо монолитных 240) — экспоненциальное ускорение
2. **Тривиальная жёсткая квантизация** — `sign(x)` вместо поиска ближайшего из 240
3. **Иерархическое масштабирование** — от 8 до 4096 точек без потери структуры
4. **Связь с теорией кодирования** — коды Рида-Маллера, преобразование Уолша-Адамара
5. **Интерпретируемость** — каждая ось = обучаемая бинарная оппозиция

Главный вопрос тот же, что и для E8: **даёт ли предопределённая геометрическая структура преимущество перед learned кодбуком?** Ответ требует эксперимента. Но если E8 заслуживает proof-of-concept, то И-Цзин заслуживает его не меньше — с более сильными вычислительными аргументами.
