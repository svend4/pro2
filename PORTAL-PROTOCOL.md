# Nautilus Portal Protocol

**Смысловой MCP-протокол для взаимодополняющих репозиториев**

---

## Основная идея

Репозитории не сливаются. Они остаются независимыми — как Word-документ, PDF и Excel-таблица остаются разными форматами. Портал — это Office Suite: умеет открывать все форматы, показывать их вместе, находить связи между ними.

```
┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐
│  info1   │  │   pro2   │  │   meta   │  │  repo-N  │
│ .info1   │  │  .pro2   │  │  .meta   │  │ .custom  │
└────┬─────┘  └────┬─────┘  └────┬─────┘  └────┬─────┘
     │              │              │              │
     └──────────────┴──────────────┴──────────────┘
                           │
                   ⬡ NAUTILUS PORTAL
                   (читает все форматы)
```

## Два варианта использования

### Вариант A — Локальный портал (portal.py)

```bash
python portal.py --query "crystal"    # поиск во всех репо
python portal.py --html > report.html # HTML-отчёт
python portal.py --serve              # веб-сервер :8000
python portal.py --json               # машиночитаемый вывод
```

**Когда использовать:** исследование, локальная разработка, быстрый поиск.

### Вариант B — Автономный репозиторий `svend4/nautilus`

Отдельный репо с порталом как самостоятельным продуктом:
```
svend4/nautilus/
├── README.md          ← главная страница портала
├── portal.py          ← движок
├── adapters/          ← адаптеры для каждого репо
│   ├── info1.py
│   ├── pro2.py
│   └── meta.py
├── passports/         ← паспорта репозиториев
│   ├── info1.md
│   ├── pro2.md
│   └── meta.md
└── index.html         ← статический веб-портал (GitHub Pages)
```

**Когда использовать:** публичный портал, GitHub Pages, точка входа для новых участников.

## Протокол подключения репозитория

Чтобы репозиторий стал совместимым с Nautilus Portal, нужно **три вещи**:

### 1. Файл `nautilus.json` в корне репо

```json
{
  "name": "info1",
  "version": "1.0",
  "format": "info1",
  "description": "Методологическая система параллельного двунаправленного развития",
  "native_unit": "Markdown-документ с α-уровнем",
  "abstraction_model": "alpha_levels",
  "entry_points": {
    "concepts": "02-Информационная-система/",
    "methodology": "README.md#methodology",
    "corpus": null
  },
  "bridges": {
    "pro2": "Карточки ↔ Концепты Q6",
    "meta": "α-уровни ↔ CA-энтропия"
  },
  "tags": ["methodology", "documentation", "cards", "alpha-levels"]
}
```

### 2. Файл `PASSPORT.md` — паспорт репозитория

(см. шаблон ниже)

### 3. Необязательно: адаптер `nautilus_plugin.py`

```python
from nautilus_base import BaseAdapter

class MyRepoAdapter(BaseAdapter):
    name = "my-repo"

    def fetch(self, query: str) -> list[PortalEntry]:
        # логика поиска в своём репо
        ...

    def describe(self) -> dict:
        # описание формата
        ...
```

## Шаблон PASSPORT.md

```markdown
# Паспорт репозитория: [NAME]

## Что это
[1-2 предложения: суть проекта]

## Какую грань реальности исследует
[Угол зрения: методологический / семантический / символьный / ...]

## Нативный формат
- Единица хранения: [карточка / концепт / правило / ...]
- Пространство: [α-уровни / Q6 / CA-правила / ...]
- Объём: [количество единиц]

## Чем полезен другим репозиториям
- Даёт: [что экспортирует]
- Принимает: [что импортирует]

## Как подключиться
\`\`\`bash
# установить адаптер
pip install nautilus-portal
nautilus add svend4/[name]
\`\`\`

## Мосты
| К репо | Тип моста | Пример |
|--------|-----------|--------|
| pro2   | семантический | Карточки → Q6-концепты |
| meta   | символьный    | α-уровень → CA-энтропия |
```

## Семантический MCP-протокол

**MCP** здесь = **Meaning Connection Protocol** (протокол смысловых связей).

Не технический API, а **соглашение о смысле**:

```
Репо A знает: "кристалл — это порядок из хаоса"  (meta, CA-правило 30)
Репо B знает: "crystal → Q6[0,1,1,0,1,0]"        (pro2, эмбеддинг)
Репо C знает: "Кристалл — карточка α=-2"          (info1, реализация)

Портал говорит: все три описывают одно — "структура возникающая из правил"
```

Консенсус достигается не через единую БД, а через **согласованные теги и мосты**.

---

## Уровни совместимости

| Уровень | Что нужно | Что даёт |
|---------|-----------|----------|
| **0 — Обнаруживаемый** | `nautilus.json` | Портал находит репо |
| **1 — Читаемый** | `PASSPORT.md` | Портал показывает описание |
| **2 — Связанный** | `bridges` в json | Портал строит граф связей |
| **3 — Интерактивный** | `nautilus_plugin.py` | Полнотекстовый поиск через адаптер |

Каждый репо сам выбирает нужный уровень. Уровень 0 занимает 5 минут.

## Философия

> Репозитории — как слепые мудрецы, ощупывающие слона.
> Каждый знает свою часть досконально.
> Портал — не тот, кто знает всё,
> а тот, кто слышит всех и переводит между ними.

Nautilus Portal не хранит знания. Он **маршрутизирует смыслы**.
