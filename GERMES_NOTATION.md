## 1. Алфавит и размерности

### 1.1. Визуальные примитивы (shirina/dlina/visota)

```
1.) shirina (ширина)   — точки:       . . . . . . . . . . . . . .
2.) dlina  (длина)     — горизонтали: = = = = = = = =
                                      - - - - - - - -
                                      _____________
3.) visota (высота)    — вертикали:   | | | | |
```

Три измерения пространства → три типа линий. Это **геометрическая
основа**: ширина, длина, высота задают три оси Q3.

### 1.2. Тернарный алфавит {+, <, −}

```
+++ = 444   (всё ян,     полная активность)
<<< = 333   (всё нейтр., равновесие)
--- = 222   (всё инь,    полная пассивность)
... = 111   (пустота,    нуль)
```

Четыре уровня: {`...`, `---`, `<<<`, `+++`} = {0, −1, 0, +1}?
Нет: это **четырёхзначная шкала**, где `<` — нейтраль между `−` и `+`.

Соответствие с `pro2`:
```
Germes +  →  +1  (ян)
Germes <  →   0  (нейтраль, STE straight-through)
Germes -  →  -1  (инь)
Germes .  →   ?  (пустота, маска/padding)
```

### 1.3. Фигуры 1,2,3-мерные

Каталог визуальных символов — **графемы для форм пространства**:

```
s c   — кривые (sphere, curve)
z v   — углы (zigzag, vertex)
w x   — волна, крест (wave, cross)
> <   — направления (arrow right/left)
o     — окружность (circle)
/ \ ( ) — диагонали, дуги
! i   — вертикаль с точкой (emphasis, i-point)
```

## 2. Структура Q3: 8 состояний, 6 + 2

### 2.1. Полный список состояний

Три бинарных оси {+, −}³ → 2³ = **8 состояний**:

```
Чистые (вес 0 и 3):
  ---   (все −)   = вода, пустота
  +++   (все +)   = огонь, полнота

Смешанные с весом 1 (один +):
  --+
  -+-
  +--

Смешанные с весом 2 (два +):
  -++
  +-+
  ++-
```

### 2.2. Гексагон — 6 «срединных» состояний

Исключая два «чистых» (`---`, `+++`), получаем **6 вершин гексагона**:

```
             Node 1
            --+  (вес 1)
           /    \
    Node 6       Node 2
    ++-           -+-
(вес 2)          (вес 1)
    |    ·   ·   |
    Node 5       Node 3
    +-+           +--
(вес 2)          (вес 1)
           \    /
            Node 4
            -++  (вес 2)
```

Рёбра трёх цветов:
- **Синие** (outer ring): `--+` — `++-` — `+-+` — `-++` — `-+-` — `+--` — обратно
- **Зелёные** (inner ring): соединения через 1 шаг
- **Красные** (диагонали через центр): противоположные вершины

Расстояние Хэмминга: все соседи в гексагоне отличаются ровно на **2 бита**
(это вершины октаэдра, проецированного на плоскость).

## 3. Семантическое заземление гексагона

Каждый узел несёт **три семантических слоя**: природное явление,
животное, боевое движение:

| Узел | Код | Природа | Животное | Единоборство |
|------|-----|---------|----------|--------------|
| 1 | `--+` | молния быстрая | журавль / ветви | kopeo (копьё) / udar (удар) |
| 2 | `-+-` | глина горячая | тигр / ствол | blok (блок) / mech (меч) |
| 3 | `+--` | дым тёплый | леопард / корни | zahvat (захват) / kinjal (кинжал) |
| 4 | `-++` | ветер порывистый | обезьяна / листья | plashe (плащ) / brosok (бросок) |
| 5 | `+-+` | туман прохладный | крокодил / цветы | sheit (щит) / uklon (уклон) |
| 6 | `++-` | течение медленное | змея / плоды | stoika (стойка) / knut (кнут) |

**Закономерность**: природный агент (`молния`, `глина`...) + объект природы
(`ветви`, `ствол`, `корни`...) + боевая техника.

## 4. Q2 ⊂ Q3: четыре элемента

Два первых бита задают **4 классических элемента**:

```
++ = огонь / свет    →  +++  пламя жаркое
                         ++-  молния быстрая

-+ = воздух          →  +--  ветер порывистый
                         -+-  туман прохладный

+- = земля / глина   →  +-+  глина горячая
                         -++  дым тёплый

-- = вода            →  --+  течение медленное
                         ---  запруда холодная
```

Третий бит — **модификатор** внутри элемента:
```
+3 = активная форма  (+++ = пламя жаркое, +++→+++ = активный огонь)
-3 = пассивная форма (++-  = молния, активный → лёгкий)
```

**Вложение Q2 ⊂ Q3** — это прообраз Матрёшки-Квантинайзера в `pro2`.

## 5. Тернарные тригра́ммы 3×3

Расширение до тернарного алфавита {+, <, −}³ → **3³ = 27 тригра́мм**.

Визуальное представление: каждая тригра́мма — столбец из 3 строк,
каждая строка одна из {+++, <<<, ---}:

```
+++   +++   +++
+++   <<<   ---
+++   ---   <<<
(3 из 27 — "верхний" слой, где первая строка = +++)
```

Полный перебор 27 тригра́мм по слоям:

| Слой | Строка 1 | Строки 2,3 | Число |
|------|----------|-----------|-------|
| Ян  | `+++` | все 9 комбинаций | 9 |
| Нейтраль | `<<<` | все 9 комбинаций | 9 |
| Инь | `---` | все 9 комбинаций | 9 |

27 тригра́мм = ternary Q3. Это **промежуточный уровень** между
бинарным Q3 (8 состояний) и полным Q6 (64 гексаграммы).

## 6. Место Germes в иерархии проекта

```
Germes Q3 (бинарный):    8 состояний  = 2³
Germes Q3 (тернарный):  27 тригра́мм   = 3³
Матрёшка:                4 атома      = 2²    ← Q2
PseudoRAG:              16 архетипов  = 2⁴    ← Q4
NautilusMoME:           36 пар        = 6²
YiJing / pro2:          64 гексаграммы = 2⁶   ← Q6
```

Germes — **исходный документ**: он создан без компьютера, чисто
концептуально, и содержит в зародыше всё, что проект реализует
программно:

| Концепт в Germes | Реализация в `pro2` |
|------------------|---------------------|
| {+, <, −} алфавит | Ternary logic: {+1, 0, −1} |
| 6-узловой гексагон | 6 линий гексаграммы |
| 4 элемента (Q2) | 2 свободных бита в Q6 |
| Матрёшка Q2 ⊂ Q3 | Матрёшка-Квантинайзер Q2 ⊂ Q4 ⊂ Q6 |
| Семантика узлов | Онтология 64 гексаграмм И-Цзин |
| Тернарные тригра́ммы | 3⁶ = 729 (нереализованный уровень) |

### 6.1. Что Germes предсказал

1. **Тернарная логика**: `<` как нейтраль — это STE (straight-through estimator) в дискретизации нейронной сети
2. **Гексагональная топология**: 6 смешанных состояний, соединённых рёбрами = граф Q3 без полюсов
3. **Иерархическое вложение**: Q2 ⊂ Q3 предвосхищает Q2 ⊂ Q4 ⊂ Q6
4. **Мультимодальность**: природа + тело + действие = три семантических оси на одном узле

### 6.2. Что Germes не предусмотрел

- Дифференцируемость: ∂Loss/∂hex ≠ 0 — появится только с градиентным спуском
- Q6 вместо Q3: удвоение глубины (3→6 линий) для 64 гексаграмм
- Контрастивное обучение: Font3/Font4 = два шрифта для одного символа

## 7. Связь с SOLAN

SOLAN алфавит — **цифровое продолжение** той же идеи:

```
Germes (аналог):  рисованный символ ↔ состояние Q3
SOLAN  (цифровой): 6 бинарных сегментов ↔ 64 состояния Q6
```

В Germes визуальное сходство символов отражает близость состояний.
В SOLAN это формализовано через расстояние Хэмминга и Gray-код.

## 8. Детальный анализ гексагона: движение по кубу Q3

### 8.1. Топологические данные: расстояния Хэмминга

Шесть узлов гексагона и их коды:

```
Node 1: --+  (вес 1)  молния быстрая   / журавль / копьё-удар
Node 2: -+-  (вес 1)  глина горячая    / тигр    / блок-меч
Node 3: +--  (вес 1)  дым тёплый       / леопард / захват-кинжал
Node 4: -++  (вес 2)  ветер порывистый / обезьяна/ плащ-бросок
Node 5: +-+  (вес 2)  туман прохладный / крокодил/ щит-уклон
Node 6: ++-  (вес 2)  течение медленное/ змея    / стойка-кнут
```

Матрица расстояний Хэмминга (d):

```
     1  2  3  4  5  6
  1: 0  2  2  1  1  3
  2: 2  0  2  1  3  1
  3: 2  2  0  3  1  1
  4: 1  1  3  0  2  2
  5: 1  3  1  2  0  2
  6: 3  1  1  2  2  0
```

Три типа пар:
- **d=1** (6 пар): (1,4),(1,5),(2,4),(2,6),(3,5),(3,6) — рёбра куба
- **d=2** (6 пар): (1,2),(1,3),(2,3),(4,5),(4,6),(5,6) — диагонали граней
- **d=3** (3 пары): **(1,6),(2,5),(3,4)** — пространственная диагональ куба

### 8.2. Центр гексагона — диагональ Касаткина

Центральная точка гексагона НЕ является узлом Q3.
Это **точка (0,0,0)** — середина главной диагонали куба:

```
  ---  ←──────────────→  +++
(-1,-1,-1)  ЦЕНТР  (+1,+1,+1)
запруда холодная   пламя жаркое
 (полный инь)      (полный ян)
```

Диагональ Касаткина = ось трансформации, ось «потенциала».
Она не является состоянием — это **путь** между двумя полюсами.

Расстояния от узлов до полюсов:
```
Node 1,2,3 (вес 1): до --- = 1 шаг,  до +++ = 2 шага
Node 4,5,6 (вес 2): до --- = 2 шага, до +++ = 1 шаг
```

Гексагон — «экватор» куба. Полюса (`---` и `+++`) — «полюса».
Все три красных антипода (1↔6, 2↔5, 3↔4) пересекают центр по
направлению этой диагонали.

### 8.3. Два треугольника — две триграммы

Гексагон содержит два **правильных треугольника** (все стороны d=2):

```
   ЗЕЛЁНЫЙ треугольник ▲ (вес-1, ближе к ---)
   Узлы: 1 (--+), 2 (-+-), 3 (+--)
   Все попарные расстояния = 2
   Семантика: молния, глина, дым — «пробуждение», начало ян

   ЖЁЛТЫЙ треугольник ▽ (вес-2, ближе к +++, «вниз», плохо виден)
   Узлы: 4 (-++), 5 (+-+), 6 (++--)
   Все попарные расстояния = 2
   Семантика: ветер, туман, течение — «закрепление», зрелость ян
```

Эти два треугольника образуют **Звезду Давида / Гексаграмму И-Цзин**:
- ▲ зелёный = нижняя триграмма (основание ян)
- ▽ жёлтый  = верхняя триграмма (завершение ян)

Вершины зелёного треугольника **антиподальны** вершинам жёлтого:
```
Node 1 (--+)  ←d=3→  Node 6 (++-) ← красная линия
Node 2 (-+-)  ←d=3→  Node 5 (+-+) ← красная линия
Node 3 (+--) ←d=3→  Node 4 (-++) ← красная линия
```
«Другие стороны зелёного и жёлтого треугольника — другие вершины»:
каждый узел ▲ проецируется сквозь центр (Касаткинскую диагональ)
в соответствующий узел ▽.

### 8.4. Четыре цвета рёбер = четыре типа движения

#### Жёлтые рёбра — d=2, внутри одного треугольника
Движение «на месте»: **перераспределение** без набора/потери ян.

```
Внутри ▲: --+ ↔ -+- ↔ +--  (меняем 2 бита, вес остаётся 1)
Внутри ▽: -++ ↔ +-+ ↔ ++-  (меняем 2 бита, вес остаётся 2)
```

Смысл: «скачок через одну ступеньку» — адаптация внутри уровня,
специализация, боковая эволюция. Стрессовый манёвр: два бита
меняются одновременно, промежуточного состояния нет.

#### Синие рёбра — d=1, между треугольниками
Одиночный бит флиппуется: **рождение / творение**.

```
(1,4),(1,5),(2,4),(2,6),(3,5),(3,6)
Пример: --+ → -++ (бит 2 флиппуется: +1 ян)
        молния быстрая → ветер порывистый
```

Это единственные рёбра, которые СУЩЕСТВУЮТ в кубе Q3 как его рёбра.
Шаг +1 ян = включение одного нового измерения = «рождение».
Шаг -1 ян = выключение = «смерть» или «возврат к истоку».

Это также единственный **обратимый** манёвр без потери информации:
зная конечную точку d=1, всегда можно вернуться.

#### Зелёные рёбра — d=2, между треугольниками (или тот же тип что жёлтые?)
В диаграмме Гермеса зелёный цвет присваивается тому же классу d=2,
но для межтреугольного случая... или это другой набор рёбер.

*Уточнение:* два отдельных треугольника в диаграмме — зелёный (▲)
и жёлтый (▽) — и есть «jump over 1 step». Зелёные рёбра = стороны
зелёного треугольника, жёлтые рёбра = стороны жёлтого треугольника.

#### Красные рёбра (пунктир) — d=3, через центр
**Все три бита меняются одновременно.** Самый радикальный прыжок.

```
1↔6: --+ ↔ ++-  молния ↔ течение медленное (максимальный контраст!)
2↔5: -+- ↔ +-+  глина ↔ туман прохладный
3↔4: +-- ↔ -++  дым ↔ ветер порывистый
```

«Деградация и разрушение себя ИЛИ рождение себя»:
- Разрушение: полное обращение состояния = «смерть текущего я»
- Рождение: но после полного распада = место для нового = «феникс»
- Это И-Цзиновские пары 否/泰: предел инь рождает ян, предел ян рождает инь

Почему «3 или 4 шага» в кольце? В кольце из 6 узлов (1-2-3-6-5-4):
- 3 шага по кольцу = противоположная вершина (кратчайший путь)
- 4 шага по кольцу = та же вершина в обход (длинный путь)
- Оба ведут к одному: d=3 в пространстве Хэмминга

### 8.5. Полная схема движений в Q3

```
              ---  (запруда холодная, полный инь)
             / | \
     синий  /  |  \  синий
           /   |   \
       Node1  Node2  Node3
      (--+)  (-+-)  (+---)
        ↑      ↑      ↑
  вес-1 │ зел. ▲ │ молния/глина/дым
        └──────┴──────┘
  зелёные стороны треугольника (d=2)
        .
        .  ← ЦЕНТР (Касаткинская диагональ)
        .
  жёлтые стороны треугольника (d=2)
        ┌──────┬──────┐
  вес-2 │ жел. ▽ │ ветер/туман/течение
       ↓      ↓      ↓
      Node4  Node5  Node6
      (-++)  (+-+)  (++-)
           \   |   /
     синий  \  |  /  синий
             \ | /
              +++  (пламя жаркое, полный ян)
```

Красные диагонали пробивают систему СКВОЗЬ центр:
```
Node1 ←──────── центр ────────→ Node6  (молния ↔ течение)
Node2 ←──────── центр ────────→ Node5  (глина ↔ туман)
Node3 ←──────── центр ────────→ Node4  (дым ↔ ветер)
```

### 8.6. Семантическая интерпретация движений

#### Синее движение (d=1) — «Рождение»
Добавляется один ян. Боевой аналог:
- `копьё/удар` (1) → `плащ/бросок` (4): атака переходит в обволакивающий бросок
- `блок/меч` (2) → `плащ/бросок` (4): блок переходит в захват и бросок
- `захват/кинжал` (3) → `щит/уклон` (5): фиксация переходит в подвижное уклонение

Природный аналог: молния (--+) → ветер (-++): вспышка рождает движение.

#### Жёлтые/зелёные рёбра (d=2) — «Адаптация»
Смена специализации без смены уровня. Боевой аналог:
- `копьё` ↔ `блок` ↔ `захват`: три разных способа использовать одинь ян.
- `плащ` ↔ `щит` ↔ `стойка`: три разных способа удержать два ян.

Природный аналог: молния ↔ глина ↔ дым (одинаковая «квантовая» энергия,
разные «физические» формы).

#### Красное движение (d=3) — «Революция»
Полное обращение. Боевой аналог:
- `копьё/удар` ↔ `стойка/кнут`: удар в пустоту → кнут-удержание
- `захват/кинжал` ↔ `плащ/бросок`: контроль ↔ свобода

Природный аналог: молния быстрая ↔ течение медленное —
предельный контраст, смерть-и-рождение.

### 8.7. Связь с И-Цзин: Q3 как прообраз Q6

Гексагон Гермеса — это **8 триграмм И-Цзин**, спроецированных на плоскость:

```
Germes:         И-Цзин:
---  (полюс инь)   = ☷ Кунь (чистая земля/вода)
--+  (вес 1)       = ☳ Чжэнь (гром)
-+-  (вес 1)       = ☵ Кань (вода)
+--  (вес 1)       = ☶ Гэнь (гора)
-++  (вес 2)       = ☴ Сунь (ветер)
+-+  (вес 2)       = ☲ Ли (огонь)
++-  (вес 2)       = ☱ Дуй (озеро)
+++  (полюс ян)    = ☰ Цянь (чистое небо/огонь)
```

Две триграммы (верхняя + нижняя) = одна гексаграмма. Поэтому
переход Q3 → Q6 = переход от триграмм к гексаграммам.

В Q6 каждый «синий шаг» d=1 = изменение одной линии гексаграммы.
В И-Цзин это называется «изменяющаяся линия» (変爻, biàn yáo).

### 8.8. Касаткинская диагональ как ось обучения

В контексте нейросетей `pro2`:
```
---  = нулевой вектор, начало обучения, отсутствие знания
+++  = насыщенный вектор, полное знание, предельный ян

Синие шаги (d=1) = gradient descent, постепенное уточнение
Красные прыжки (d=3) = catastrophic forgetting / breakthrough insight
Центр = "среднее" представление, многозначное, нейтральное
```

Касаткинская диагональ в Q6 = 6-битное пространство, которое
модель должна «освоить» через синие шаги, избегая красных прыжков.
