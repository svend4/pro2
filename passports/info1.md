# Паспорт репозитория: info1

## Что это

Методологическая система параллельного двунаправленного развития (⇑⇓↔). 74 документа, 1156 связей. Восемь уровней абстракции — от онтологии (α=+4) до исходного кода (α=-4). Не хранилище знаний, а **структура для мышления**.

## Какую грань реальности исследует

**Методологический угол**: как организовать мышление и документацию чтобы одновременно двигаться вниз (к реализации) и вверх (к обобщению), не теряя связей между уровнями.

## Нативный формат

- **Единица хранения**: Markdown-документ с α-уровнем (-4..+4)
- **Пространство**: 9 уровней абстракции × N доменов
- **Объём**: 74+ документов, 1156 связей
- **Структура директорий**: каждая директория = уровень или домен

### Схема α-уровней:

```
α=+4  Онтология        ← самые абстрактные понятия
α=+3  Философия
α=+2  Методология      ← этот репо живёт здесь
α=+1  Руководства
α= 0  Концепция        ← точка пересечения
α=-1  Спецификации
α=-2  Реализация
α=-3  Примеры
α=-4  Код              ← конкретный исходный код
```

## Чем полезен другим репозиториям

**Даёт:**
- Структурированные тексты с известным уровнем абстракции
- 74 размеченных документа как обучающий корпус
- Методологию двунаправленного развития (⇑⇓) как метапаттерн
- Карточки как атомарные единицы знания (аналог концептов Q6)

**Принимает:**
- Семантические эмбеддинги своих документов (из pro2)
- Символьную верификацию паттернов (из meta)
- Оценку внутренней согласованности документов

## Мосты к другим репо

| К репо | Тип моста | Механизм |
|--------|-----------|----------|
| **pro2** | Семантический | α-уровень → глубина Q6 (α+4)/8 * 6 → dim |
| **meta** | Структурный | Иерархия α ↔ иерархия CA-энтропий |
| **nautilus** | Портальный | `Info1Adapter` в `portal.py` |

### Соответствие α ↔ Q6:

```
info1 α=+4  ↔  pro2 Q6 dim=5  (самое абстрактное измерение)
info1 α=+2  ↔  pro2 Q6 dim=3
info1 α= 0  ↔  pro2 Q6 dim=2  (нейтральная точка)
info1 α=-2  ↔  pro2 Q6 dim=1
info1 α=-4  ↔  pro2 Q6 dim=0  (самое конкретное)
```
