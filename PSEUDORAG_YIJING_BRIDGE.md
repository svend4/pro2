## 1. Центральный тезис

Оба проекта независимо пришли к одной идее:

> **Реальность (и язык, описывающий её) организована вокруг конечного набора
> архетипических категорий, которые живут на гиперкубе.**

```
PseudoRAG:        {0,1}⁴  →  16 информационных архетипов
YiJing-Transformer: {-1,+1}⁶  →  64 гексаграммы (архетипа мышления)
```

Разница — в **разрешении** (16 vs 64) и **методе** (эвристика vs обученная проекция).

Это не два разных проекта. Это **два уровня одной системы**.

## 2. Формальное вложение Q4 ⊂ Q6

### 2.1. Изоморфизм пространств

```
Q4 = {0,1}⁴   ← PseudoRAG (16 точек)
Q6 = {-1,+1}⁶ ← YiJing (64 точки)
```

Отображение Q4 → Q6 через замену {0→-1, 1→+1} и добавление 2 нулевых координат:

```
φ: Q4 → Q6
φ(b₁, b₂, b₃, b₄) = (2b₁-1, 2b₂-1, 2b₃-1, 2b₄-1, 0, 0)
```

Это вложение переводит 16 PseudoRAG-архетипов в 16 «экваториальных» точек Q6
(с нулями в последних двух координатах).

Остальные 48 гексаграмм = **тонкая дифференциация** внутри каждой категории.

### 2.2. Метрическая совместимость

Расстояние Хэмминга между двумя PseudoRAG-архетипами:
```
d_Ham(MSCO, MDCO) = 1  (отличается только S↔D)
```

Расстояние Хэмминга между соответствующими гексаграммами в Q6:
```
d_Ham(φ(MSCO), φ(MDCO)) = 1  (тот же бит)
```

**Метрика сохраняется.** Семантическая близость архетипов PseudoRAG = семантическая
близость гексаграмм YiJing в первых 4 битах.

## 3. Сравнение архетипических осей

### Оси PseudoRAG (4 бита)

| Бит | Ось | -1 / 0 | +1 / 1 | Смысл |
|-----|-----|---------|---------|-------|
| b₁ | Materiality | A (Абстрактное) | M (Материальное) | Онтологический статус |
| b₂ | Dynamics | S (Статичное) | D (Динамичное) | Временно́й характер |
| b₃ | Scale | E (Элементарное) | C (Комплексное) | Структурная сложность |
| b₄ | Structure | O (Упорядоченное) | F (Текучее) | Степень организации |

### Оси YiJing (6 бит)

| Бит | Триграммная позиция | Инь (-1) | Ян (+1) |
|-----|---------------------|----------|---------|
| b₁ | Нижняя триграмма, линия 1 | инь | ян |
| b₂ | Нижняя триграмма, линия 2 | инь | ян |
| b₃ | Нижняя триграмма, линия 3 | инь | ян |
| b₄ | Верхняя триграмма, линия 1 | инь | ян |
| b₅ | Верхняя триграмма, линия 2 | инь | ян |
| b₆ | Верхняя триграмма, линия 3 | инь | ян |

### Гипотетическое соответствие осей

```
PseudoRAG Materiality  ↔  YiJing b₁ (базовая полярность — инь/ян)
PseudoRAG Dynamics     ↔  YiJing b₂ (изменение — 变爻)
PseudoRAG Scale        ↔  YiJing b₄ (переход нижнее/верхнее)
PseudoRAG Structure    ↔  YiJing b₃ (внутренняя организация триграммы)
                       +   YiJing b₅, b₆ (дополнительная тонкость, нет в PseudoRAG)
```

Это гипотеза. Верификация — через семантическую карту (Фаза 2 Варианта 3).

## 4. Таблица соответствий: 16 → 64

Каждый PseudoRAG-архетип соответствует кластеру из **4 гексаграмм** (фиксированы
биты b₁–b₄, свободны b₅–b₆).

| PseudoRAG | Имя | Гексаграммы Q6 (b₅b₆ варианты) | Хэмминг-расстояние внутри |
|-----------|-----|----------------------------------|---------------------------|
| MSEO | Кристалл | 4 гексаграммы | 1–2 |
| MSEF | Песок | 4 гексаграммы | 1–2 |
| MSCO | Здание | 4 гексаграммы | 1–2 |
| MSCF | Лес | 4 гексаграммы | 1–2 |
| MDEO | Механизм | 4 гексаграммы | 1–2 |
| MDEF | Организм | 4 гексаграммы | 1–2 |
| MDCO | Машина | 4 гексаграммы | 1–2 |
| MDCF | Город | 4 гексаграммы | 1–2 |
| ASEO | Аксиома | 4 гексаграммы | 1–2 |
| ASEF | Архетип | 4 гексаграммы | 1–2 |
| ASCO | Теория | 4 гексаграммы | 1–2 |
| ASCF | Культура | 4 гексаграммы | 1–2 |
| ADEO | Алгоритм | 4 гексаграммы | 1–2 |
| ADEF | Интуиция | 4 гексаграммы | 1–2 |
| ADCO | Программа | 4 гексаграммы | 1–2 |
| ADCF | Общество | 4 гексаграммы | 1–2 |

**Интерпретация:** Если «Здание» (MSCO) — это архетип «сложная, упорядоченная,
материальная, статичная структура», то 4 гексаграммы внутри этого кластера —
это **нюансы**: «здание в момент строительства», «здание как наследие»,
«здание как символ», «здание как утилита».

## 5. Что умеет PseudoRAG, чего нет у YiJing

### 5.1. Работающая семантическая проекция

`QueryExpander.calculate_archetype_relevance()` — это **уже рабочий алгоритм**
проекции текста на архетипы:

```python
# Из pseudorag/core/query_expander.py
def calculate_archetype_relevance(self, archetype, topic_data):
    relevance = archetype.default_priority / 5.0        # базовый prior
    keyword_matches = sum(1 for kw in keywords if kw in query)
    relevance += keyword_matches * 0.1                  # keyword signal
    relevance += domain_bonus.get(archetype.code, 0)   # domain prior
    return min(relevance, 1.0)
```

Это можно использовать как **учитель** для `HexagramProjection` в YiJing:
- Инициализировать веса проекции так, чтобы Q4-паттерны уже работали
- Затем дообучить на Q6 (64 гексаграммы)

### 5.2. Шаблонные вопросы — готовая структура знаний

850 структурированных вопросов для любой темы — это:
- Обучающий сигнал для архетипного reasoning
- Тест «правильности» гексаграммной проекции
- Инструмент создания датасета для тонкой настройки

### 5.3. Интерпретируемые метки

Каждый архетип PseudoRAG имеет:
- Человекочитаемое имя («Город», «Кристалл», «Алгоритм»)
- Ключевые слова
- Примеры сущностей

Это **золотая разметка** для семантической карты Варианта 3 (Фаза 2).

## 6. Что умеет YiJing, чего нет у PseudoRAG

### 6.1. Дифференцируемость

YiJing-Transformer обучается градиентным спуском. PseudoRAG — эвристика.

```
PseudoRAG: text → keyword rules → archetype  (детерминированно, не обучается)
YiJing:    text → h ∈ R^d → HexagramProjection → softmax(64)  (обучается)
```

### 6.2. Тонкость 64 vs 16

16 архетипов не различают «здание как наследие» и «здание как утилита».
64 гексаграммы — различают. Это не избыточность, а **необходимая глубина**.

### 6.3. Топологическая динамика (BianGuaTransform)

PseudoRAG: статичная классификация (архетип = метка).
YiJing: динамическая навигация (архетип → следующий архетип через 变卦).

```
YiJing может описать ПРОЦЕСС:
MSCO (Здание) --[одна линия]--> MDCO (Машина)  ← здание стало живым
MSCO (Здание) --[одна линия]--> ASCF (Культура) ← здание стало символом
```

### 6.4. Тернарное третье состояние

{-1, 0, +1}⁶: ноль = «в переходе» (变爻). У PseudoRAG нет переходных состояний.

## 7. Roadmap интеграции

### Этап 0 — Валидация соответствия (1 неделя)

**Задача:** Подтвердить, что Q4-архетипы PseudoRAG действительно вложены в Q6.

```python
# Эксперимент:
# 1. Взять 1000 предложений с разметкой PseudoRAG-архетипов
# 2. Прогнать через HexagramProjection (YiJing)
# 3. Проверить: предложения одного PseudoRAG-архетипа
#    должны давать гексаграммы из одного кластера (4 близкие точки)

def validate_q4_in_q6(sentences_with_labels, hexagram_model):
    results = {}
    for sentence, pseudorag_label in sentences_with_labels:
        hex_distribution = hexagram_model.project(sentence)  # (64,)
        top_hexagrams = hex_distribution.topk(4).indices
        results[pseudorag_label].append(top_hexagrams)

    # Метрика: насколько кластеризованы гексаграммы внутри одного PseudoRAG-архетипа?
    clustering_score = compute_intra_cluster_hamming(results)
    return clustering_score  # ожидается: < 2.5 (близкие гексаграммы)
```

**Ожидаемый результат:** Предложения одного PseudoRAG-архетипа дают гексаграммы
с Хэмминг-расстоянием ≤ 2 (один кластер из 4).

---

### Этап 1 — PseudoRAG как учитель (2 недели)

**Задача:** Использовать PseudoRAG для инициализации и supervision YiJing.

```
Обучение в два этапа:

Шаг A (Distillation):
  Учитель: PseudoRAG.calculate_archetype_relevance()  → вектор (16,)
  Ученик:  HexagramProjection.forward()               → вектор (64,)
  Loss:    KL(P_student_aggregated, P_teacher)
           (агрегировать 64 в 16 через кластерную принадлежность, затем KL)

Шаг B (Fine-tuning на языке):
  Стандартный language model loss
  HexagramProjection теперь инициализирована знаниями PseudoRAG
```

**Результат:** HexagramProjection с семантически значимыми кластерами с первого
шага, без долгого самостоятельного «открытия» структуры.

---

### Этап 2 — Расширение шаблонов до 64 (2–3 недели)

PseudoRAG генерирует ~850 вопросов из 16 шаблонных наборов.
YiJing может расширить это до **64 наборов** — по одному на каждую гексаграмму.

```
Текущая структура PseudoRAG:
  MSCO → 6 шаблонов вопросов

Расширение до Q6:
  MSCO + (00) → «Здание как утилита»:      вопросы о функции и эффективности
  MSCO + (01) → «Здание как наследие»:     вопросы о сохранении и истории
  MSCO + (10) → «Здание как символ»:       вопросы о значении и представлении
  MSCO + (11) → «Здание в становлении»:    вопросы о строительстве и изменении
```

Это **четырёхкратное увеличение разрешения** онтологии.

---

### Этап 3 — Обратная связь: YiJing улучшает PseudoRAG (месяц)

После обучения YiJing на Q6 — использовать выученные представления для
улучшения PseudoRAG:

```
YiJing.HexagramProjection.embed(text) → вектор в R^6 (мягкие биты)
    ↓ first 4 bits
PseudoRAG relevance scores (learned, не эвристические)
```

Результат: PseudoRAG с **нейросетевым ядром** вместо keyword-matching.

## 8. Архитектура объединённой системы

### 8.1. Схема

```
╔══════════════════════════════════════════════════════════════════╗
║             ОБЪЕДИНЁННАЯ СИСТЕМА: PseudoRAG + YiJing             ║
╠══════════════════════════════════════════════════════════════════╣
║                                                                  ║
║  УРОВЕНЬ 0: Пользовательский запрос                              ║
║  "Расскажи о транспорте в Берлине"                               ║
║       ↓                                                          ║
║  УРОВЕНЬ 1: Q4 — PseudoRAG (грубая онтология)                   ║
║  MDCO=0.7, MDCF=0.6, ADCF=0.4 ...                              ║
║       ↓  (supervision signal для обучения)                       ║
║  УРОВЕНЬ 2: Q6 — YiJing HexagramProjection (тонкая онтология)   ║
║  hex#23=0.12, hex#35=0.09, hex#11=0.08 ...                      ║
║       ↓                                                          ║
║  УРОВЕНЬ 3: BianGuaAttention (навигация по Q6)                   ║
║  [транспорт] → MDCO → hex#35 → [Прогресс] ← переход            ║
║       ↓                                                          ║
║  УРОВЕНЬ 4: ArchetypalInterlingua (посредник)                    ║
║       ↓                                                          ║
║  УРОВЕНЬ 5: Генерация текста                                     ║
║  "Берлин имеет развитую систему метро (U-Bahn, 9 линий)..."      ║
║                                                                  ║
╠══════════════════════════════════════════════════════════════════╣
║  ПАРАЛЛЕЛЬНО: QueryExpander (PseudoRAG)                         ║
║  "Транспорт в Берлине" → 72 структурированных вопроса           ║
║  → валидация: ответы модели покрывают ли все вопросы?           ║
╚══════════════════════════════════════════════════════════════════╝
```

### 8.2. Роль каждого проекта

| Аспект | PseudoRAG | YiJing-Transformer | Объединение |
|--------|-----------|---------------------|-------------|
| Онтология | Q4 (16 категорий) | Q6 (64 категории) | Q4 ⊂ Q6 (иерархия) |
| Метод | Эвристика (правила) | Градиентный спуск | Distillation |
| Выход | Список вопросов | Текстовая генерация | Вопросы + ответы |
| Интерпретируемость | Высокая (метки) | Средняя (гексаграммы) | Q4-метки для Q6 |
| Обучаемость | Нет | Да | PseudoRAG как prior |

### 8.3. Метрики совместной работы

| Метрика | Измерение |
|---------|-----------|
| **Q4-кластеризация** | Предложения одного PseudoRAG-архетипа → Хэмминг ≤ 2 среди топ-4 гексаграмм |
| **Template coverage** | % шаблонных вопросов PseudoRAG, на которые модель генерирует релевантный ответ |
| **Archetype consistency** | При перефразировке запроса доминирующий архетип меняется в пределах кластера |
| **Distillation accuracy** | KL(Q4_pseudorag, Q4_yijing_aggregated) < 0.1 |

## Заключение

PseudoRAG и YiJing-Transformer — не конкурирующие проекты, а **двухуровневая система**:

```
PseudoRAG (Q4):       грубый компас    — 16 направлений, человекочитаемо
YiJing (Q6):          точный компас    — 64 направления, дифференцируемо

Вместе:               навигационная система с двумя разрешениями
```

Интеграция позволяет:
1. **PseudoRAG → YiJing:** Инициализация, supervision, интерпретируемые метки
2. **YiJing → PseudoRAG:** Нейросетевое ядро взамен keyword-matching
3. **Обоюдно:** Валидация семантической структуры на двух уровнях детализации
