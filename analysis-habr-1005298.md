# Анализ статьи: «Geometry > Scale: Как 40М параметров на решётке E8 обходят классические трансформеры»

**Источник:** https://habr.com/ru/articles/1005298/
**Автор:** bootstraptor
**Дата публикации:** 2 марта 2026
**Дата анализа:** 5 марта 2026

---

## 1. Краткое содержание

Автор представляет проект **Lila-E8** — языковую модель с 40 млн параметров, в которой стандартный механизм self-attention заменён на архитектуру, основанную на решётке исключительной группы Ли E8. Утверждается, что модель достигает результатов, сопоставимых или превосходящих модели с 60М+ параметров (в частности, упоминается сравнение с моделью Microsoft).

### Заявленные метрики:
| Метрика | Значение |
|---------|----------|
| Параметры | 40 млн |
| Train Loss | 0.37 |
| Validation Loss | 0.44–0.53 |
| Контекстное окно | 512 токенов (удерживает 750+) |

---

## 2. Ключевые тезисы автора

1. **Стандартный attention создаёт избыточность** — «трение» в латентном пространстве, которое мешает эффективному обучению.
2. **Решётка E8 — оптимальная упаковка сфер в 8 измерениях** — используется как структура для организации латентного пространства, минимизируя «информационную вязкость».
3. **Модель «резонирует» со структурой языка**, а не запоминает паттерны.
4. **Geometry > Scale** — эффективность через геометрические принципы, а не через масштабирование.

---

## 3. Критический анализ

### 3.1. Сильные стороны

- **Оригинальная идея.** Применение алгебраических структур (группы Ли, решётки) к архитектуре нейросетей — направление с реальным научным потенциалом. Есть прецеденты: equivariant neural networks, geometric deep learning (работы Бронштейна и др.).
- **Открытый код.** Проект доступен на GitHub под лицензией AGPLv3 с notebook для воспроизведения — это плюс для верифицируемости.
- **Фокус на эффективности.** Идея уменьшения параметров при сохранении качества актуальна в контексте растущих затрат на обучение LLM.

### 3.2. Слабые стороны и вопросы

- **Недостаточная методология сравнения.** Утверждение «чище, чем у 60М-модели от Microsoft» — расплывчатое. Не указано: какая именно модель, на каком бенчмарке, с какими гиперпараметрами проводилось сравнение.
- **Отсутствие стандартных бенчмарков.** Нет результатов на общепринятых бенчмарках (GLUE, SuperGLUE, MMLU, HellaSwag и т.д.). Train/validation loss — недостаточная метрика для заявлений о превосходстве.
- **Малое контекстное окно.** 512 токенов — крайне малый контекст по современным стандартам. Утверждение об «удержании 750+ токенов без зацикливания» нуждается в формальной верификации.
- **Разрыв train/val loss.** Разница 0.37 vs 0.44–0.53 может указывать на переобучение (overfitting), что противоречит тезису об эффективности геометрического подхода.
- **Нечёткая терминология.** Выражения «информационная вязкость», «резонирует со структурой языка» — метафоры, не математические определения. Это снижает научную строгость.
- **Масштабируемость не доказана.** 40М параметров — это micro-scale. Непонятно, сохраняются ли преимущества при масштабировании до 1B+ параметров.

### 3.3. Потенциальные риски интерпретации

- Статья может создать впечатление, что решётка E8 — «серебряная пуля» для замены transformer-архитектуры. На практике для таких заявлений нужны peer-reviewed публикации и воспроизведение результатов независимыми командами.
- Сравнение с конкретной моделью Microsoft без указания названия и условий эксперимента — признак cherry-picking.

## 4. Контекст: решётка E8 в математике

Решётка E8 — это действительно замечательный математический объект:
- Оптимальная упаковка сфер в 8 измерениях (доказано Вязовской, 2016, Филдсовская премия 2022).
- 240 ближайших соседей у каждого узла.
- Связана с теорией струн, кодами исправления ошибок, и другими областями математики.

Однако **сам по себе факт оптимальности упаковки в R⁸ не гарантирует** оптимальности для представления языковых структур. Необходимо теоретическое обоснование, почему именно эта геометрия подходит для NLP-задач.

## 6. Глубокий технический анализ исходного кода

Проведён анализ репозитория [SPUTNIKAI/sovereign-lila-e8](https://github.com/SPUTNIKAI/sovereign-lila-e8). Ниже — разбор ключевых архитектурных компонентов.

### 6.1. Генерация корней E8 (`models/model.py`)

Реализация математически корректна:

- **Тип 1 (112 корней):** Все перестановки `(±1, ±1, 0, 0, 0, 0, 0, 0)` — выбор 2 из 8 координат × 4 комбинации знаков = C(8,2)×4 = 112.
- **Тип 2 (128 корней):** Все `(±½)⁸` с чётным числом минусов — 2⁸/2 = 128.
- **Итого:** 240 корней — математически верно.
- **Верификация:** Проверяются нормы (√2), уникальность (240), центрирование (сумма ≈ 0). Всё правильно.

```python
# Проверка свойств — строгая и корректная
assert roots.shape == (240, 8)
norms = torch.norm(roots, dim=1)
assert torch.abs(norms - math.sqrt(2.0)).max() < 1e-6
```

### 6.2. E8Quantizer — мягкая квантизация

```python
def forward(self, x):
    distances = x_norm_sq - 2 * cross + roots_norm_sq  # L2 расстояния
    weights = F.softmax(-distances / self.temp, dim=-1)  # soft-assignment
    quantized = weights @ roots                           # взвешенная сумма
    return x + (quantized - x).detach()  # straight-through estimator
```

**Анализ:**
- Используется **straight-through estimator** (STE): в forward pass возвращается `quantized`, но градиенты проходят через `x`. Это стандартный приём из VQ-VAE (van den Oord et al., 2017).
- Температура `temp=0.1` — довольно жёсткая, при ней softmax почти вырождается в argmax. Это означает, что на практике каждый вектор сильно привязывается к одному корню E8.
- **Проблема:** STE создаёт несогласованность градиентов — forward идёт по квантованным значениям, а backward — по непрерывным. При `temp=0.1` разрыв может быть существенным.

### 6.3. E8Attention — геометрический bias

```python
q_proj = torch.einsum('bhtd,hd->bht', q_e8, self.head_dirs)  # проекция на E8-направление
k_proj = torch.einsum('bhtd,hd->bht', k_e8, self.head_dirs)
geo_bias = q_proj.unsqueeze(-1) * k_proj.unsqueeze(-2)         # ранг-1 bias матрица
scores = scores + head_scales * geo_bias
```

**Анализ:**
- Каждая голова attention получает **одно направление** из 240 корней E8 (первые `n_heads=8` корней).
- Геометрический bias — это **ранг-1 матрица** (внешнее произведение скалярных проекций). Это добавляет всего 1 степень свободы на голову.
- `head_scales` инициализирован нулями → модель начинает как стандартный трансформер и **постепенно учит**, нужна ли геометрия. Это хороший инженерный приём.
- **Критика:** Только первые 8 из 240 корней используются для attention. Выбор первых 8 — это тип-1 корни `(±1, ±1, 0⁶)`, что не покрывает тип-2 структуру. Нет обоснования, почему именно эти 8 оптимальны.

### 6.4. E8TransformerLayer — архитектура слоя

```
Input → LayerNorm → E8Attention → Residual
      → LayerNorm → Linear(d→8) → E8Quantizer → Linear(8→d) → Scale → Residual
      → LayerNorm → FFN(d→4d→d) → Residual
```

**Анализ:**
- **Бутылочное горлышко 8D:** Каждый слой проецирует `d_model=512` в 8 измерений, квантует, и проецирует обратно. Это **экстремальное сжатие** (коэффициент 64x). Большая часть информации неизбежно теряется.
- **E8 strength = 0.01:** Масштабирующий коэффициент очень мал. Вклад E8-ветки в residual stream ≈ 1% от основного сигнала. Это фактически означает, что **E8-компонент является слабой регуляризацией**, а не основным вычислительным путём.
- **Шум при обучении:** Добавлен гауссов шум (`0.001` и `0.005`) — дополнительная регуляризация, но без теоретического обоснования конкретных значений.
- **Resonator закомментирован:** `E8GraphResonator` (ассоциативная память на графе E8) — заявлен в README, но **отключён в коде** (`#self.resonator`).

### 6.5. E8GraphResonator — ассоциативная память

```python
def encode_relation(self, node_a_idx, node_b_idx, relation_weight=1.0):
    relation = torch.outer(root_a, root_b)
    self.reasoning_gate.add_(relation * relation_weight)
```

**Анализ:**
- Классическая **Hopfield-like ассоциативная память** через внешние произведения.
- Проекция `(240, 8) → (240, d_model)` через случайную матрицу — по теореме Джонсона-Линденштраусса сохраняет расстояния, но **теряет алгебраическую структуру E8**. Корни E8 после проекции в 512D уже не являются системой корней.
- **Не используется в основной модели** — закомментирован в `E8TransformerLayer`.

### 6.6. Конфигурация и обучение

| Параметр | Значение | Комментарий |
|----------|----------|-------------|
| d_model | 512 | Стандартно для малых трансформеров |
| n_layers | 12 | Достаточно глубокая сеть |
| n_heads | 8 | Стандартно |
| block_size | 512 | Малое контекстное окно |
| vocab_size | 2048 | Очень малый словарь (SentencePiece) |
| lr | 5e-5 | Консервативный learning rate |
| batch_size | 4 | Очень малый батч |
| total_steps | 150K | Относительно немного |
| dropout | 0.05 | Умеренный |
| e8_strength | 0.01 | **Очень слабый E8-вклад** |

**Критика обучения:**
- Нет learning rate scheduler (warmup, cosine decay) — это нестандартно и может ограничивать качество.
- `batch_size=4` при `block_size=512` = 2048 токенов на шаг — очень мало, высокая дисперсия градиентов.
- Нет gradient accumulation для компенсации малого батча.
- Датасет **TinyStories** — синтетический датасет простых детских рассказов. Loss 0.37 на TinyStories ≠ loss на естественном тексте.

### 6.7. Подсчёт параметров

Оценка параметров модели:
- `tok_emb`: 2048 × 512 = **1.05M**
- `pos_emb`: 1 × 512 × 512 = **0.26M**
- Каждый слой:
  - `qkv`: 512 × 1536 = 0.79M
  - `out`: 512 × 512 = 0.26M
  - `to_e8` + `from_e8`: 512×8 + 8×512 = 0.008M
  - `ffn`: 512×2048 + 2048×512 = 2.1M
  - `LayerNorm` × 3: ~0.003M
  - **Итого на слой: ~3.16M**
- 12 слоёв: **~37.9M**
- `head`: 512 × 2048 = **1.05M**
- **Общий итог: ~40.3M** — совпадает с заявленным.

**Ключевое наблюдение:** E8-специфичные параметры (`to_e8`, `from_e8`, `head_scales`, `e8_scale`) составляют **< 0.3% от общего числа параметров**. Модель на 99.7% — обычный трансформер.

## 7. Главные выводы технического анализа

### 7.1. E8 — регуляризация, а не архитектура

Вопреки названию и позиционированию, E8-компоненты играют роль **слабой регуляризации**, а не замены attention:
- `e8_strength = 0.01` — E8-ветка вносит ~1% в residual
- `head_scales` начинаются с 0 — геометрический bias изначально отключён
- Основной вычислительный путь — стандартный трансформер (attention + FFN)

### 7.2. Заголовок вводит в заблуждение

Утверждение «обходят классические трансформеры» не подтверждается кодом. Модель **является** классическим трансформером с небольшой добавкой. Это ближе к «трансформер с E8-регуляризацией» или «E8-augmented transformer».

### 7.3. Справедливость сравнения

- Обучение на **TinyStories** (простой синтетический текст) с **vocab_size=2048** делает метрики несопоставимыми с моделями, обученными на естественных корпусах.
- Отсутствие LR scheduler, малый батч — указывают на неоптимальную конфигурацию обучения, что может занижать baseline и завышать относительный вклад E8.

### 7.4. Что действительно интересно

- Идея **soft quantization к фиксированным геометрическим точкам** как регуляризация латентного пространства — заслуживает исследования. Это похоже на VQ-VAE, но с предопределённой структурой кодбука.
- **Ранг-1 геометрический bias в attention** — простой и элегантный, но нужен ablation study чтобы доказать, что именно E8 (а не любые случайные направления) дают эффект.
- Подход «начни с нуля и постепенно учи вклад E8» (zero-init scales) — правильное инженерное решение.

### 7.5. Что нужно для убедительности

1. **Ablation study:** Заменить E8-корни на (a) случайные точки на сфере, (b) вершины других правильных многогранников, (c) learned кодбук. Если разницы нет — E8 не причём.
2. **Стандартные бенчмарки:** Результаты на общепризнанных задачах, не только loss на TinyStories.
3. **Честный baseline:** Обычный трансформер с той же конфигурацией, тем же датасетом, тем же числом шагов и оптимизированным lr schedule.
4. **Масштабирование:** Показать, что E8-вклад растёт при увеличении модели, а не остаётся на уровне 1%.

## 8. Итоговая оценка (обновлённая)

| Аспект | Оценка | Комментарий |
|--------|--------|-------------|
| Оригинальность идеи | Высокая | Применение E8 к NLP — нестандартно |
| Математическая корректность E8 | Высокая | Генерация 240 корней верна |
| Научная строгость | Низкая | Нет ablation, нет бенчмарков, метафорическая терминология |
| Архитектурная честность | Низкая | Заявлено «замена attention», по факту — слабая добавка |
| Инженерное качество кода | Средняя | Чистый код, но отсутствует scheduler, малый батч, закомментированные компоненты |
| Воспроизводимость | Средняя | Код открыт, но нет подробного описания воспроизведения метрик |
| Практическая значимость | Не доказана | TinyStories + vocab 2048 — не production-сценарий |
| Потенциал | Есть | Но нужна серьёзная экспериментальная работа |

**Финальный вердикт:** Проект представляет собой стандартный 40М-трансформер с E8-регуляризацией, вносящей ~1% в вычисления. Математика E8 реализована корректно, но её влияние на качество модели не доказано. Для подтверждения заявлений необходим строгий ablation study, сравнивающий E8 с другими геометрическими структурами и random baselines.
