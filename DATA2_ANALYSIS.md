## Применение 2: Архитектурные улучшения из scarab_algorithm.py

### 2.1 ScarabQuaternion → Улучшенный LCI

Текущий LCI = `(1 - |w_A - w_B|) · π` — это приближение.  
Оригинал Крюкова — кватернионная норма:

```python
# scarab_algorithm.py → ScarabQuaternion
|A| = √(BVS² + SVS² + MVS² + ChVS²) = π  # at mastery
```

Применение к нашей архитектуре:
```python
def lci_quaternion(w_abstract, w_dynamic, w_concrete, w_bridge):
    """LCI как норма кватерниона 4 сфер.
    BVS=Abstract, SVS=Dynamic, MVS=Concrete, ChVS=Bridge (точка пересечения).
    """
    return math.sqrt(w_abstract**2 + w_dynamic**2 + w_concrete**2 + w_bridge**2) * math.pi
```

**Эффект:** Более точное измерение резонанса, учитывающее BidirBridgeExpert.

### 2.2 DualMatchStickAutomaton = Q6

`DualMatchStickAutomaton` в scarab_algorithm.py оперирует **64 символами** из 6-битных комбинаций — это в точности Q6! Расстояние перехода = Hamming-расстояние (1-2 бита).

Вывод: `DualMatchStickAutomaton` — это дискретная реализация нашего `Q6ExpertBank`. Переходная граф MSA с диаметром=3 идентичен свойствам Q6 (диаметр=6, но в подпространстве = 3).

### 2.3 Параметр деформации k → Формализация асимметрии обучения

Деформированная восьмёрка с параметром k:
```
y(t) = B·[sin(2ωt + π/2) + (k-1)/(k+1)·sin(ωt)]
k=1: симметричная ∞
k=7: несимметричная (петля A в 7× больше)
```

**Текущее:** `_SERIES_A=7, _SERIES_B=1` — это точно k=7!  
**Формализация:** добавить параметр `k` к `figure8_hmoe_v4`, вычислять теоретическую долю A: `p_A = k/(k+1) = 7/8 = 0.875`.

### 2.4 Правила Скарабея → Mapping на архитектуру

| Правило Скарабея | Аналог в HierarchicalMoE | Статус |
|------------------|--------------------------|--------|

### 3.2 Спектральная матрица ETD → Цель обучения (VOLUME_179)

4×4 матрица синергии ETD имеет собственные значения: `{π/8, π/8, π/8, 5π/8}`  
- Доминирующий вектор (1,1,1,1)/2 = равновесие = LCI=π
- След матрицы = π = закон сохранения Нётер!
- Спектральный зазор = 5π/8 - π/8 = π/2 → мера стабильности

**Применение:** Матрица весов роутинга {w_A, w_X, w_B, w_bridge} должна стремиться к состоянию (1/4, 1/4, 1/4, 1/4) — именно этот вектор = доминирующий собственный вектор ETD-матрицы.

### 3.3 Пропускная способность канала (VOLUME_157)

ETD параметры: B = 4/π Гц, SNR = π  
C = (4/π)·log₂(1+π) ≈ 2.63 бит/шаг

Это ограничение на информацию, которую роутер может передавать за один шаг. Текущий routing_entropy ≈ 1.1 бит < 2.63 бит → роутер работает на 42% пропускной способности.  
**Цель:** довести routing_entropy до ≥2.63 бит.

### 3.4 Камуфляж/угроза → BidirBridgeExpert (VOLUME_07)

## Мнение и оценка данных

### Плюсы репозитория data2

1. **Математическая глубина**: 200+ томов продвинутой математики (теория групп, TDA, инфогеометрия) — лучший источник для ABSTRACT кластера (Theory/Models).

2. **Прямая связь с архитектурой**: `scarab_algorithm.py` — это альтернативная реализация той же топологии. ScarabQuaternion ≡ LCI, DualMatchStickAutomaton ≡ Q6ExpertBank.

3. **Уникальность**: ETD Крюкова — оригинальная теория, аналогов мало. Тренировка на этих данных даст уникальные концептуальные связи.

4. **Объём**: 405 файлов × ~10-50 кБ ≈ 2-4 ГБ чистого текста — существенное дополнение к корпусу.

### Минусы / риски

1. **Нумерологическая составляющая**: VOLUME_62, VOLUME_64, VOLUME_157 содержат "нечётные числа везде" как самоцель. Это не математика — это нарратив. При обучении модель может усвоить нумерологический паттерн, а не математическую структуру.

2. **Повторение**: Многие тома повторяют одни и те же 7 аксиом ETD с минимальными вариациями. Нужна дедупликация.

3. **Спекулятивность**: Утверждения типа "trace=π = закон Нётер" — аналогия, не теорема. Модель должна различать строгие математические результаты и авторские интерпретации.

4. **Смешение уровней**: Vol 1-50 (прикладные) и Vol 201-310 (чистая математика) требуют разных токенизаций и контекстов.

### Варианты использования (предложения)

**Вариант A: Прямое добавление в корпус**
```
Theory-кластер += [VOLUME_07, VOLUME_15, VOLUME_36, VOLUME_41, VOLUME_42,

```
Результат: модель получает знания ETD + математику как тренировочный сигнал.

**Вариант B: Концептуальный импорт (без тренировки)**
Взять конкретные идеи (ScarabQuaternion LCI, параметр k, нечётные слои) и реализовать в архитектуре.  
Преимущество: нет риска "замусоривания" корпуса нумерологией.

**Вариант B (рекомендуется)**: Использовать оба варианта — тренировать на отфильтрованных томах (математика, AI) и реализовать конкретные концепции (ScarabQuaternion LCI, k-параметр).
