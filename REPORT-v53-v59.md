## 1. Обзор проекта

YiJing-Transformer — трансформерная архитектура с индуктивным смещением на основе геометрии гиперкуба Q6 = {-1,+1}⁶. Ключевая идея: 64 гексаграммы И-Цзин = вершины 6-мерного гиперкуба, что даёт естественную дискретную структуру для кодбука, attention-масок и квантизации.

### Семь источников

| # | Автор | Вклад |
|---|-------|-------|
| 1 | Склярова | Дворцовая структура (8 дворцов × 8 гексаграмм), порядки Фуси/Вэнь-вана |
| 2 | Фомюк | D4-эквивариантность, антиподальная симметрия |
| 3 | Андреев | Треугольная матрица, привилегированная ось |
| 4 | Касаткин | Диофантовы координаты куба, кубическая диагональ, дуальное встраивание |
| 5 | Герман | Теория упаковок P=2^k (теоретическое обоснование) |
| 6 | Беляев | Рычажные весы → Heisenberg attention + FlowerGAT |
| 7 | SOLAN | 76-глифный алфавит, Q6 Glyph Ecosystem |

## 2. Хронология версий

### v51–v52: Фундамент
- Реализованы все 7 ступеней интеграции (32 подступени)
- 13 новых конфиг-параметров, 41 тест
- Модули интегрированы в `model.py`, геометрия вынесена в пакет `geometry/`

### v53: Первый бенчмарк источников
- **Подключены 8 «мёртвых» модулей** к forward pass
- **Три бенчмарка:** WikiText (LM), S-box (crypto), pairwise combinations

### v54: Anti-interference routing
- Kasatkin 3D embedding
- Три-направленный бенчмарк

### v55: Convergence Bridge
- Гибридная иерархия: глифы ↔ токены
- GlyphComposer: 4-уровневая композиция (glyph → edge → spectral → sigil)
- TokenAbstractor: мягкая кластеризация на 64 архетипа

### v56: TernaryQuantizer + MatrixGrammar
- Расширение до {-1, 0, +1}⁶ (тернарная квантизация)
- MatrixGrammar: автоматы Лукасевича + 变爻 (Atamiri)

### v57: AbrialeLayer
- Событийно-управляемые изотропные N-местные связи (Пацкин)
- Commit-gate: порог 0.01 → разреженная активация

### v58: Bridge of Modules
- Иерархическая cross-attention медиация между источниками
- LightweightBridge: рабочая интеграция в YiJingGPT
- Расширение до 7 источников

### v59: Финальный бенчмарк
- 5 архитектур + 7 ablation = 12 конфигов
- Систематическое сравнение на синтетическом WikiText (800 шагов, d=128)

## 4. SOLAN: Алфавит и глифы — статус

### 4.1. Что реализовано

**GlyphTokenizer** (`tokenizer/glyph_tokenizer.py`) — полностью рабочий:
- 76 символов SOLAN (10 цифр + 26 uppercase + 26 lowercase + 14 спецсимволов)
- Каждый глиф = вершина Q6, 6 визуальных сегментов (верх, низ, лево, право, диаг1, диаг2)
- `encode(text)` → Tensor shape (seq_len, 6), значения {-1, +1}
- `decode(vertices)` → текст через nearest-neighbor в Хэмминг-метрике
- `contrastive_pairs()` → позитивные пары (flip 1 бит) для контрастивного обучения
- 64 из 76 символов покрывают все вершины Q6; 12 — дублирующие с модификацией

**GlyphComposer** (`models/geometry/convergence.py`):
- 4-уровневая иерархия: glyph → edge → spectral → sigil
- Граф-лапласиан для оконных спектральных сигнатур

**TokenAbstractor** (`models/geometry/convergence.py`):
- Мягкая кластеризация на 64 архетипа (= 64 гексаграммы)
- Temperature annealing
- Корреляция с I-Ching архетипами

**Теорема 15** (`theoretical-foundations.md`):
- Формальное доказательство инъективности φ: Alphabet → Q6
- Hamming distance = визуальное различие глифов
- Gray-code размещение на 8×8 сетке

### 4.3. Выводы по алфавиту

SOLAN-76 реализован как **standalone модуль**: токенизация, декодирование, контрастивные пары — всё работает. Однако он **не интегрирован в основной training loop** — модель по-прежнему использует `CharTokenizer`. Глифы остаются теоретической конструкцией с доказанным изоморфизмом, но без практической валидации в бенчмарках.

## 6. Рекомендации

### Приоритет 1: Lean Model
Оставить только Heisenberg + FlowerGAT + AbrialeLayer + Bridge. Удалить 5 «вредных» источников. Ожидаемый результат: PPL < 1.07 при ~1M параметров вместо ~1.9M.

### Приоритет 2: Валидация на реальных данных
Текущие бенчмарки — на синтетическом WikiText (800 шагов, маленький размер). Нужна проверка на полном WikiText-103 или OpenWebText.

### Приоритет 3: Интеграция SOLAN
Подключить GlyphTokenizer к основному пайплайну. Проверить, даёт ли визуальный изоморфизм Q6 преимущество над стандартным CharTokenizer.

### Приоритет 4: Масштабирование
Проверить, сохраняется ли преимущество abriale_bridge при d=256, d=512 и на более длинных последовательностях.
