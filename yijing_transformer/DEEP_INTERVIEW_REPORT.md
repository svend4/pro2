## Метаданные
- **Дата**: 2026-03-13
- **Модель**: NautilusMoME (3,024,852 params, step 5000)
- **Токенизатор**: BPE 4096

## 1. ЗНАКОМСТВО — КТО ТЫ?

### Английский промпт
**Prompt**: `# About Me\n\nI am a language model called`
**Routing**: MATH=0.228 | SYSTEM=0.221 | INFO=0.216
**Ответ**: Коллаж из pytest-паттернов, русского текста, случайных фрагментов
**PPL на "I am a language model called NautilusMoME"**: 452.0

**Мой анализ**: Модель не имеет понятия о своём имени. PPL 452 = этот текст для неё почти инопланетный. Интересно, что routing распределился равномерно (MATH/SYSTEM/INFO) — модель "не знает", какого эксперта активировать для саморефлексии.

### Русский промпт
**Prompt**: `# Описание модели\n\nЯ — нейросеть, которая`
**Routing**: RECON=0.655 | INFO=0.212
**Ответ**: Поток полубессвязного русского текста с философскими обрывками

**Мой анализ**: RECON доминирует (0.655) — модель чётко определила: "это русский текст". Но генерация — это не ответ, а продолжение в стиле обучающих данных. Модель "говорит", но не "отвечает".

### Кодовый промпт
**Prompt**: `class NautilusMoME:\n    """A Mixture of Micro-Experts that`
**Routing**: RECON=0.313 | MATH=0.180 | INFO=0.159

**Мой анализ**: Даже в формате кода модель не может осмысленно описать себя. Интересно, что RECON лидирует, а не CODE — видимо, docstring ассоциируется с документацией.

## 4. ЭКСПЕРТНАЯ МАРШРУТИЗАЦИЯ — РАБОТАЕТ!

Ключевые паттерны:
- **Python код** → CODE=0.261 (лидирует)
- **React/JS** → CODE=0.296 (лидирует)
- **SQL** → SYSTEM=0.361 (системные запросы!)
- **Русский текст** → RECON=0.559 (мощная активация!)
- **K8s YAML** → SYSTEM=0.327 + INFO=0.310
- **Math формула** → MATH=0.255 + SYSTEM=0.271
- **Dockerfile** → INFO=0.339
- **pytest assert** → CODE=0.299 + MATH=0.293

**Мой анализ**: Маршрутизация — это **главное достижение** NautilusMoME. Эксперты реально специализировались:
- CODE активируется на Python/JS/React
- RECON "владеет" русским текстом
- SYSTEM обрабатывает SQL, инфраструктуру, формулы
- INFO — конфигурационные файлы

Однако уверенность маршрутизации всего 2-3% — эксперты слишком "мягко" активируются. Нужен более агрессивный top-k cutoff.

## 5. ПРЕДСКАЗАНИЯ МОДЕЛИ

| Промпт | Top-1 предсказание | Правильно? |
|--------|-------------------|------------|
| `import torch.nn as ` | `yn` | Нет (нужно `nn`) |
| `def __init__(self, ` | `root` | Частично (валидный Python) |
| `assert isinstance(result, ` | `list` | **ДА!** |
| `if __name__ == ` | `"` | **ДА!** |
| `from collections import ` | `defaultdict` | **ДА!** |
| `const [...] = useState(` | `()` | Частично |
| `except Exception as e:` | `return` | Нет (нужно `raise`/`print`) |
| `# TODO: ` | `#` | Нет |

**Результат**: 3/8 точных, 2/8 частично правильных, 3/8 неверных.

**Мой анализ**: Для 3M параметров — **впечатляюще**. Модель знает:
- `assert isinstance(x, list)` — самый частый паттерн из pytest
- `if __name__ == "` — каноничная конструкция
- `from collections import defaultdict` — самый популярный импорт

Ошибки тоже показательны: `import torch.nn as yn` — модель путает `nn` с `asyncio` (видимо `syn`→`syncio`).

## 8. ИТОГОВАЯ ОЦЕНКА CLAUDE

### Что модели нравится (её словами, через routing):
- **Код** — это её мир. PPL 8-15 на Python/React/pytest
- **Импорты и структуры** — каноничные паттерны вроде `if __name__`
- **Числа и формулы** — Math-эксперт работает

### Что модели не нравится:
- **Естественный язык** — она не для этого создана
- **Саморефлексия** — PPL 452 на описание себя
- **Вне-доменные тексты** — медицина, право, поэзия

### Плюсы архитектуры:
1. **Маршрутизация работает** — CODE, RECON, SYSTEM специализировались
2. **Модульность** — эксперты можно дообучать по отдельности
3. **Эффективность** — 59 tok/s, только 2/6 экспертов активны
4. **Масштабируемость** — архитектура готова к 10x-100x увеличению

### Минусы:
1. **Генерация — коллаж** — после 15-20 токенов всё смешивается
2. **Нет instruction following** — только продолжение текста
3. **Domain mixing** — русский + код + тесты в одном ответе
4. **Vocab 4096** — мало для естественного языка
5. **Контекст 256** — слишком короткий для сложных задач

### Рекомендации по улучшению:
1. Масштабирование до 10-50M параметров
2. Vocabulary 16K-32K
3. Контекст 2048+ токенов
4. Instruction tuning (минимальный — даже 1K примеров поможет)
5. Более агрессивный top-k в маршрутизации
6. ArchetypeLayer в inference (не только training)
7. DPO/RLHF для alignment

### Оценки:
| Аспект | Оценка |
|--------|--------|
| Как исследовательский проект | **8/10** |
| Как архитектурная идея | **9/10** |
| Как текстовый генератор | **3/10** |
| Как proof-of-concept MoE | **9/10** |
| Как образовательный инструмент | **8/10** |

### Главный вывод:

NautilusMoME — это **успешный proof-of-concept**, доказывающий что:
- MoE-архитектура работает при 3M параметрах
- Expert routing обучается реальной специализации
- Модульный подход жизнеспособен для микро-моделей
- BPE + sparse activation — эффективная комбинация

Это не "плохая GPT" — это **исследовательский инструмент** в другой категории. Сравнивать его с ChatGPT так же бессмысленно, как сравнивать прототип двигателя с готовым автомобилем. Ценность — в архитектурных идеях, которые масштабируются.
