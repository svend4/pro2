# Cross-Domain Analogy: Превращение "коллажа" в достоинство

## Исходная идея (автор — пользователь)

> Коллаж из фрагментов разных доменов — не недостаток, а нереализованный
> потенциал. Как в науке: есть биология, есть физика, а есть **биофизика** —
> где формулы из одной науки описывают явления другой.
>
> Пословицы и крылатые выражения — это "гуманитарные формулы":
> концентрированная мудрость в нескольких словах. Модель уже имеет
> информацию, но не умеет ей пользоваться. Нужны специальные
> микромодули для осмысленного кросс-доменного переноса.

## Архитектурное решение: CrossDomainAnalogy

### Компоненты

```
core_first → Router → 6 Experts → NautilusBridge
                                        ↓
                               [CrossDomainAnalogy]  ← NEW
                                        ↓
                                   core_second
```

**1. ProverbCondenser** — сжимает вывод эксперта в "пословицу"
- Attention-weighted pooling с обучаемым query
- Спрашивает: "в чём суть того, что этот эксперт видит?"
- Результат: один вектор-формула на эксперта

**2. AnalogyPair** — один мостик между двумя доменами (15 пар для 6 экспертов)
- Проецирует "пословицы" в общее пространство аналогий
- Измеряет силу аналогии (similarity gate)
- Синтезирует кросс-доменный инсайт

**3. CrossDomainAnalogy** — координатор
- Активирует аналогии только когда сила > порога
- Суммирует инсайты и добавляет к bridge output

### Конкретные пары и их семантика

| Пара | Аналогия | Пример |
|------|----------|--------|
| MATH↔HUMAN | Математика психотипов | 64 гексаграммы ↔ 16 MBTI × 4 функции |
| MATH↔CODE | Алгоритмическая формализация | Теорема → тест → доказательство |
| CODE↔SYSTEM | Паттерн → архитектура | Design pattern ↔ system design |
| HUMAN↔INFO | Этика → организация знаний | Мудрость ↔ каталогизация |
| CODE↔RECON | Паттерн распознавания → код | OCR → API → автоматизация |
| MATH↔RECON | Геометрия → реконструкция | Q6 гиперкуб → пазл-сборка |

## Результаты эксперимента

### Что сработало
- Модуль корректно встраивается между bridge и core_second
- Все 15 пар аналогий вычисляются
- Градиенты проходят через весь pipeline

### Что ожидаемо не сработало (без обучения)
- Все пары имеют одинаковую силу (~0.50) — sigmoid без обучения
- Генерация не изменилась — gate=0.05 слишком мал для необученного модуля
- Нет дифференциации "полезных" vs "бесполезных" аналогий

## Что нужно для полной реализации

### Фаза обучения CrossDomainAnalogy

**Phase 4: Analogy Training** (после Phase 3 — router fine-tuning):
1. Заморозить core + experts + router + bridge
2. Обучать только CrossDomainAnalogy на смешанных данных
3. Loss = основной + auxiliary analogy loss:
   - Поощрять дифференциацию (не все пары одинаковой силы)
   - Поощрять контекстную зависимость (разные промпты → разные аналогии)
   - Penalize collapse (когда все аналогии = 0 или все = 1)

### Данные для обучения аналогий

Идеальные обучающие примеры — тексты на **пересечении** доменов:
- MATH↔CODE: математические алгоритмы (fibonacci, sorting proofs)
- MATH↔HUMAN: I-Ching interpretations (уже есть в данных!)
- CODE↔HUMAN: ethical AI code reviews
- **Пословицы о программировании** — буквально "гуманитарные формулы для кода"

### Ключевое улучшение: Asymmetric Analogy

Текущая реализация: A↔B симметрична.
Нужна: A→B ≠ B→A (физика→биология ≠ биология→физика).
Это удвоит число пар (30 вместо 15), но позволит моделировать
направленный перенос знания.

## Философское обоснование

Проект Yi Jing Transformer уже имеет фундамент для этого:
- 64 гексаграммы = 64 архетипа = универсальная "интерлингва"
- Q6 гиперкуб = пространство где все домены имеют координаты
- 6 экспертов = 6 "взглядов" на один и тот же токен

CrossDomainAnalogy — это буквально реализация идеи
**"Archetypal Interlingua"** из PLAN-v51, но на уровне
micro-expert routing, а не на уровне embedding space.
