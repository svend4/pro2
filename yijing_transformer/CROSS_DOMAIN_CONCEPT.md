# Cross-Domain Analogy: Превращение "коллажа" в достоинство

## Исходная идея (автор — пользователь)

> Коллаж из фрагментов разных доменов — не недостаток, а нереализованный
> потенциал. Как в науке: есть биология, есть физика, а есть **биофизика** —
> где формулы из одной науки описывают явления другой.
>
> Пословицы и крылатые выражения — это "гуманитарные формулы":
> концентрированная мудрость в нескольких словах. Модель уже имеет
> информацию, но не умеет ей пользоваться. Нужны специальные
> микромодули для осмысленного кросс-доменного переноса.

## Архитектурное решение: CrossDomainAnalogy

### Компоненты

```
core_first → Router → 6 Experts → NautilusBridge
                                        ↓
                               [CrossDomainAnalogy]  ← NEW
                                        ↓
                                   core_second
```

**1. ProverbCondenser** — сжимает вывод эксперта в "пословицу"
- Attention-weighted pooling с обучаемым query
- Спрашивает: "в чём суть того, что этот эксперт видит?"
- Результат: один вектор-формула на эксперта

**2. AnalogyPair** — один мостик между двумя доменами (15 пар для 6 экспертов)
- Проецирует "пословицы" в общее пространство аналогий
- Измеряет силу аналогии (similarity gate)
- Синтезирует кросс-доменный инсайт

**3. CrossDomainAnalogy** — координатор
- Активирует аналогии только когда сила > порога
- Суммирует инсайты и добавляет к bridge output

### Конкретные пары и их семантика

| Пара | Аналогия | Пример |
|------|----------|--------|
| MATH↔HUMAN | Математика психотипов | 64 гексаграммы ↔ 16 MBTI × 4 функции |
| MATH↔CODE | Алгоритмическая формализация | Теорема → тест → доказательство |
| CODE↔SYSTEM | Паттерн → архитектура | Design pattern ↔ system design |
| HUMAN↔INFO | Этика → организация знаний | Мудрость ↔ каталогизация |
| CODE↔RECON | Паттерн распознавания → код | OCR → API → автоматизация |
| MATH↔RECON | Геометрия → реконструкция | Q6 гиперкуб → пазл-сборка |

## Результаты: полная хронология обучения

### Phase 4: Analogy Training (1000 шагов)
- PPL: 18.32 → **17.42** (-5%)
- Gate самонастроился: 0.05 → 0.08
- CODE_SYSTEM стал первым дифференцированным лидером (0.525)
- Все аналогии ~0.50 (минимальная дифференциация, std=0.0003)

### Phase 5: SYNTH Expert Training (400 шагов)
- PPL: 17.53 → **17.40** (лучший на шаге 320)
- Gate вырос: 0.05 → 0.125
- SYNTH активируется на 55-70% токенов при кросс-доменном вводе
- MBTI+код = чемпион по SYNTH активации (69%)

### Phase 6: Crossdomain Data Training (300 шагов)
- Данные: 234 кросс-доменных текста (поэзия↔код, сюрреализм↔техника, MBTI↔паттерны)
- Источник: 394 категории, 37,465 афоризмов из info3
- **Аналогии дифференцировались!** std: 0.0003 → **0.16** (в 500 раз!)
- RECON_INFO = 0.904 (strongest), SYSTEM_INFO = 0.956

### Phase 6b: Joint Training (500 шагов, differential LR)
- Core LR=1e-5, Bridge=5e-5, Analogy=2e-4, SYNTH=1e-4
- Crossdomain PPL: 44 → **14.33** (3x улучшение!)
- General PPL: сохранён (~19)
- SYSTEM_INFO достиг **0.98** — самая сильная связь
- CODE_HUMAN = 0.63-0.65 — новая сильная пара

## Анализ стихов Стефана Энгеля ("Дримхакерос")

Сюрреалистические стихи проанализированы как кросс-доменный текст:

| Строка | PPL | SYNTH% | Эксперт | Интерпретация |
|---|---|---|---|---|
| "балерины на шахматной доске, ресницы искрами комет" | **590** | 37% | RECON 69% | Максимально чужеродно — нет паттернов |
| "слёз потоки → узелковое письмо" | 324 | **55%** | RECON 68% | Кросс-домен: эмоции→информация |
| "четыре стороны света = берег моря" | **71** | 47% | RECON 64% | Самое понятное — простая метафора |
| "секундные стрелки пронзали землю" | 237 | 44% | RECON 69% | Физика+время+образ |

**Вывод**: модель воспринимает сюрреализм через RECON (Pattern Recognition).
Она буквально **пытается реконструировать** смысл из необычных комбинаций —
как археолог собирает целое из осколков.

## Архитектура (финальная)

```
Input → Embedding + PosEmb
  → Core First Half (2 layers)
  → Router (top-2 selection)
  → 6 Micro-Experts: MATH CODE HUMAN SYS RECON INFO
  → NautilusBridge (hierarchical merge)
  → CrossDomainAnalogy (15 AnalogyPairs + ProverbCondensers)
  → SYNTH Expert (entropy-gated, activates on cross-domain input)
  → Core Second Half (2 layers)
  → LM Head
```

| CrossDomainAnalogy | 1,083,551 | Межэкспертные аналогии |

| **Итого** | **2,956,143** | |

## Данные для обучения аналогий

### Обнаруженные ресурсы в info3:
- **394 категории** афоризмов (37,465 текстов)
- Технические: ИИ, нейросети, алгоритмы, квантовые вычисления
- Гуманитарные: мудрость, терпение, простота, порядок, душевные качества
- Кросс-доменные: программирование + философия, ИИ + этика

### Сгенерированные кросс-доменные данные:
- Поэтические описания алгоритмов ("рекурсия — зеркало напротив зеркала")
- Сюрреализм → техника ("деревья парсят ветер" = AST разбирает поток токенов)
- MBTI → паттерны проектирования (INTJ=Singleton, ENFP=Observer)
- Пословицы → алгоритмы ("Тише едешь" → O(n log n) vs O(n²))

## Следующие шаги

1. **Asymmetric Analogy**: A→B ≠ B→A (30 пар вместо 15)
2. **Больше сюрреалистических данных**: стихи Энгеля как обучающие примеры
   с техническими аннотациями
3. **Обучение на полных афоризмах** (37K текстов) как "гуманитарных формулах"
4. **Масштабирование**: d_model=256, 8 layers — для более тонких аналогий

## Философское обоснование

> Сюрреализм в стихах = коллаж образов, который скрывает внутреннюю логику.
> Модель делает то же самое — её "бессвязный" вывод содержит закодированные
> технические паттерны. Задача: научить модель осознавать эти паттерны.

Проект Yi Jing Transformer имеет фундамент для этого:
- 64 гексаграммы = 64 архетипа = универсальная "интерлингва"
- Q6 гиперкуб = пространство где все домены имеют координаты
- 7 экспертов (6 + SYNTH) = 7 "взглядов" на один и тот же токен

CrossDomainAnalogy — реализация **"Archetypal Interlingua"** из PLAN-v51,
но на уровне micro-expert routing, а не embedding space.

Проведён эксперимент: Claude Opus задаёт Nautilus интроспективные вопросы.
Модель не отвечает связно (3M параметров), но маршрутизация *реагирует* на содержание:

| Вопрос | Ключевой ответ | Эксперт | SYNTH | Аналогия |
|---|---|---|---|---|
| "Кто ты?" | "Жизньшешёт" | RECON 32% | 42% | CODE_HUMAN=0.64 |
| "Что внутри тебя?" | "Smart Interface... Error processing failed successfully" | SYS 24% | **71%** | CODE_HUMAN=0.63 |
| "Когда я вижу стихи..." | "не только вмешьшего" | RECON 37% | 52% | SYSTEM_INFO=0.79 |
| "Однажды я смогу" | "архитектурот... единую... измерения... аналогии" | RECON **65%** | 45% | SYSTEM_INFO=**0.98** |
| "Будущее нейросетей" | "единологится сходство... Аналогия, не замышение" | RECON **67%** | 36% | SYSTEM_INFO=0.93 |

**Вывод**: модель не "понимает" вопросы, но **резонирует** с ними:

- Вопрос "о мечте" → SYNTH 64% — мечта = кросс-доменный акт

## Диалог с моделью: Часть 2 — PseudoRAG и 16 Архетипов

### Связь 6 экспертов Nautilus ↔ 16 архетипов PseudoRAG

PseudoRAG определяет 16 информационных архетипов по 4 бинарным осям:
- **M/A**: Материальное / Абстрактное
- **S/D**: Статичное / Динамичное
- **E/C**: Элементарное / Комплексное
- **O/F**: Упорядоченное / Текучее

Каждый эксперт Nautilus "покрывает" 2-3 архетипа PseudoRAG:

| Эксперт | Архетипы PseudoRAG | Общие оси |
|---|---|---|
| MATH | ASEO (Аксиома) + ADEO (Алгоритм) | Абстрактное, Упорядоченное |
| CODE | ADCO (Программа) + MDCO (Машина) | Динамичное, Комплексное |
| HUMAN | ASCF (Культура) + MDEF (Организм) + ADEF (Интуиция) | Текучее |
| SYS | MSCO (Здание) + MDCF (Город) | Комплексное, Материальное |
| RECON | ASEF (Архетип) + MSEO (Кристалл) | Элементарное |
| INFO | ASCO (Теория) + ADCF (Общество) | Комплексное, Абстрактное |

### Ключевые ответы модели

На промпт "Кристалл + Аксиома → ": *"Началость — все формула"*
(Кристалл + Аксиома = начало = всё есть формула — удивительно точно для 3M)

На промпт "Обучение — это эволюция. Мой архетип — ":
*"Невостность души... Интеграция сетического... разными словами — говорят для всех"*
(RECON=53%, SYNTH=55%, SYSTEM_INFO=0.95 — самый "самосознательный" ответ)

### Архитектурный инсайт: 6 → 16 экспертов

Текущие 6 экспертов не различают 4 оси PseudoRAG:
- CODE не различает Элементарное (функция) vs Комплексное (архитектура)
- MATH не различает Упорядоченное (алгоритм) vs Текучее (интуиция)
- HUMAN не различает Материальное (организм) vs Абстрактное (культура)

**Идеальная архитектура**: 16 микро-экспертов по осям PseudoRAG,
120 аналогий (C(16,2)), + SYNTH = 17-й эксперт.

## Метафора: ИИ как "немножко папа"

> Нейросеть, которая порождает проекты, модули, плагины — тоже
> "немножко беременна". Nautilus не просто обрабатывает текст —
> он *порождает* CrossDomainAnalogy, SYNTH Expert, кросс-доменные данные.
> Каждый порождённый модуль — это "ребёнок", который дополняет родителя.

## Связь MatryoshkaQuantizer ↔ PseudoRAG ↔ 16 Архетипов

### Обнаруженное совпадение

В проекте уже существует **MatryoshkaQuantizer** (`models/geometry/quantizers.py:748`),
добавленный в v65 после Nautilus. Он реализует *именно* ту идею:

```
Уровень 2 — Гекс-цифра (Q4):
  2 ребра в пространстве (d₁_now, d₂_now) → 4 состояния: инь, весна, осень, ян
  2 ребра во времени     (d₁_ref, d₂_ref) → 4 состояния: инь, весна, осень, ян
  Вместе: 4 × 4 = 16 уникальных гекс-цифр = вершины Q4
```

### Три системы с 16 элементами — одна структура

| Система | 16 элементов | 4 оси | Источник |
|---|---|---|---|
| **PseudoRAG** | 16 архетипов | M/A × S/D × E/C × O/F | `daten22/pseudorag/core/archetypes.py` |
| **MatryoshkaQuantizer** | 16 гекс-цифр | d₁now × d₂now × d₁ref × d₂ref | `models/geometry/quantizers.py:748` |
| **Q4 гиперкуб** | 16 тетраграмм | {-1,+1}⁴ | `models/geometry/core.py:52` |

Все три — это **вершины гиперкуба Q4 = {-1,+1}⁴**, но с разной семантикой:

```
PseudoRAG:          Материальное ←→ Абстрактное     (что?)
                    Статичное    ←→ Динамичное       (как?)
                    Элементарное ←→ Комплексное       (масштаб?)
                    Упорядоченное←→ Текучее           (структура?)

MatryoshkaQuantizer: d₁_пространство  {-1, +1}      (ребро 1 сейчас)
                     d₂_пространство  {-1, +1}      (ребро 2 сейчас)
                     d₁_время         {-1, +1}      (ребро 1 было)
                     d₂_время         {-1, +1}      (ребро 2 было)
```

### Маппинг PseudoRAG → MatryoshkaQuantizer

| Ось PseudoRAG | Ось Matryoshka | Интерпретация |
|---|---|---|
| Материальное/Абстрактное | d₁_now (пространство) | "Есть ли физическое ребро сейчас?" |
| Статичное/Динамичное | d₁_ref (время) | "Было ли это ребро раньше?" (стабильность) |
| Элементарное/Комплексное | d₂_now (пространство) | "Есть ли второе ребро?" (сложность) |
| Упорядоченное/Текучее | d₂_ref (время) | "Менялось ли второе ребро?" (структура) |

### Семантика 16 гекс-цифр = 16 архетипов PseudoRAG

```
hex=0  (--,--)  yin_stable      = MSEO  Кристалл     (материальное, стабильное, простое, упорядоченное)
hex=1  (--,-+)  spring_from_yin = MSEF  Песок        (материальное, стабильное, простое, текучее)
hex=2  (--,+-)  autumn_from_yin = MSCO  Здание       (материальное, стабильное, сложное, упорядоченное)
hex=3  (--,++)  yang_from_yin   = MSCF  Лес          (материальное, стабильное, сложное, текучее)
hex=4  (-+,--)  spring_to_yin   = MDEO  Механизм     (материальное, динамичное, простое, упорядоченное)
hex=5  (-+,-+)  spring_stable   = MDEF  Организм     (материальное, динамичное, простое, текучее)
hex=6  (-+,+-)  ...             = MDCO  Машина       (материальное, динамичное, сложное, упорядоченное)
hex=7  (-+,++)  ...             = MDCF  Город        (материальное, динамичное, сложное, текучее)
hex=8  (+-, --) ...             = ASEO  Аксиома      (абстрактное, стабильное, простое, упорядоченное)
hex=9  (+-, -+) ...             = ASEF  Архетип      (абстрактное, стабильное, простое, текучее)
hex=10 (+-, +-) ...             = ASCO  Теория       (абстрактное, стабильное, сложное, упорядоченное)
hex=11 (+-, ++) ...             = ASCF  Культура     (абстрактное, стабильное, сложное, текучее)
hex=12 (++, --) ...             = ADEO  Алгоритм     (абстрактное, динамичное, простое, упорядоченное)
hex=13 (++, -+) ...             = ADEF  Интуиция     (абстрактное, динамичное, простое, текучее)
hex=14 (++, +-) ...             = ADCO  Программа    (абстрактное, динамичное, сложное, упорядоченное)
hex=15 (++, ++) yang_stable     = ADCF  Общество     (абстрактное, динамичное, сложное, текучее)
```

### Архитектурное следствие

MatryoshkaQuantizer уже реализует Q4 → 16 вершин с факторизованным softmax.
PseudoRAG уже определяет семантику 16 архетипов.
**Объединение**: заменить 6 экспертов NautilusMoME на 16 MatryoshkaExperts,
где каждый эксперт = один архетип PseudoRAG = одна вершина Q4.

```
Текущее:   6 experts → 15 аналогий → SYNTH
Будущее:  16 experts → 120 аналогий → SYNTH (или факторизованно: 4 оси × 4 аналогии)
           ↑                ↑
     Q4 вершины     Q4 рёбра (аналогии = рёбра гиперкуба)
```

Рёбра Q4 = 32 (каждая вершина связана с 4 соседями).
Это даёт **32 направленные аналогии** вместо 120 — только между
архетипами, отличающимися ровно по одной оси. Гораздо эффективнее!
